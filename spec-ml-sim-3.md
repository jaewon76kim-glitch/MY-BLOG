## 1. 선정 이유

### 1-1. 원문 근거

2부는 다음 세 가지 구체적 예제를 수식과 실제 숫자로 완결되게 제시한다.

- **기저율 오류(base rate fallacy, 294~319행)**: 암 검사 민감도 `p(양성|암)=0.8`, 위양성률 `p(양성|암 아님)=0.096`, 사전확률 `p(암)=0.01`이 주어졌을 때, 베이즈 정리와 전체 확률의 법칙으로 `p(암|양성)≈0.0777`을 유도한다. `p(A|B)`와 `p(B|A)`를 혼동하는 오류를 성적표 예제(321~341행, `p(A+|공부함)`과 `p(공부함|A+)`가 표의 숫자에 따라 서로 달라짐)로 한 번 더 짚는다.
- **생성적 분류기(343~384행)**: 오션뷰 방 예제(호텔 A/B/C의 우도 `p(오션뷰|A)=0.75` 등, 균등 사전확률)로 `p(y=c|x) ∝ p(x|y=c)p(y=c)`를 계산해 사후확률이 가장 큰 클래스를 고르는 **MAP 분류**를 유도하고, 붓꽃 예제로 일반화한다.
- **숫자게임(Number Game, 386~462행, Tenenbaum 1999)**: 가설공간 `H`(짝수 전체, 2의 거듭제곱 등) 중 데이터 `D`와 모순되지 않는 가설들에 대해 강한 표집 가정 `p(D|h)=(1/|h|)^N`으로 우도를 계산하고, `D={16}`일 때는 `h_two`가 근소 우세(`0.167` vs `0.02`)지만 `D={16,8,2,64}`로 데이터가 늘면 우도비가 약 5000:1까지 벌어지는 것을 직접 보여준다. 이어서 `N→∞`일 때 `log p(D|h)+log p(h) ≈ log p(D|h)`로 사전확률 항이 무시되어 **MAP가 MLE로 수렴**함(443~462행)을 유도한다.

세 예제 모두 반복 학습이나 수치 최적화 없이 **닫힌 형태 공식 평가만으로 정확히 재현 가능**하다는 공통점이 있어 구현 리스크가 낮다.

### 1-2. 기존 2개 앱과 겹치지 않는 이유

| | gradient-descent-lab | cnn-conv-lab | 이번 안(베이즈 추론) |
|---|---|---|---|
| 수학적 대상 | 손실함수(가중치 공간) | 이미지/특징맵(합성곱) | 확률분포·가설공간(사후확률) |
| 핵심 서사 | 반복 스텝이 최솟값으로 수렴 | 파라미터를 바꾸면 순전파 결과가 즉시 바뀜 | 데이터가 쌓일수록 사후확률이 좁아지고 특정 가설로 쏠림 |
| 상호작용 패턴 | 시작점 클릭 → 경사하강 애니메이션 | 슬라이더 → 이미지/파라미터 수 재계산 | 데이터 포인트를 하나씩 추가 → 여러 가설의 사후확률 막대그래프가 실시간으로 재분배 |

세 앱이 도메인·서사·조작 방식에서 뚜렷이 구분되며, 특히 "반복 스텝을 지켜보는" gradient-descent-lab의 애니메이션 패턴과는 근본적으로 다르다(이번 앱에는 학습률이나 수렴 판정이 없다).

### 1-3. 과거 기각 후보 재검토

- **SVM 커널트릭**: 쌍대문제 QP를 실시간으로 풀어야 하는 문제는 이번 검토에서도 해소되지 않았다. 기각 유지.
- **k-평균/GMM 애니메이션**: 웹에 유사 사례가 많다는 사유가 이번에도 유효하다고 판단. 기각 유지.
- **어텐션 가중치 시각화**: `spec-ml-sim-2.md`의 재검토 결론(학습된 임베딩 없이는 소프트맥스 스케일링의 수학적 시연에 그친다)이 여전히 유효하다고 판단. 기각 유지.
- **퍼셉트론/역전파 MLP+XOR**: `spec-ml-sim-2.md`가 지적한 대로 "최적화가 반복되며 수렴하는 것을 지켜본다"는 패턴이 `gradient-descent-lab`과 겹친다는 사유가 여전히 유효. 기각 유지.

### 1-4. "아직 검토되지 않은 유력 후보" 중 RNN/LSTM·LLM 대신 베이즈학습을 고른 이유

지침이 제시한 두 후보를 원문(9부 1601~1702행, 11부 1837~1962행)까지 상세히 읽고 비교했다.

- **RNN/LSTM(9부)**: 핵심 수식은 `∂h_T/∂h_1 = ∏ ∂h_t/∂h_{t-1}`(기울기 소실/폭발, 1627~1643행)과 LSTM의 `c_t = f_t⊙c_{t-1} + i_t⊙c̃_t`(덧셈 경로로 기울기 보존, 1645~1673행)이다. 그런데 이 "곱셈 경로는 소실되지만 덧셈 지름길은 기울기를 보존한다"는 구조는 `cnn-conv-lab` 5번째 기능(잔차연결 vs 일반 신경망의 `∂x_L/∂x_1` 비교, 층수에 따른 라인차트)이 이미 정확히 같은 수학적 골격으로 구현했다. LSTM 게이트를 추가해도 시각화의 핵심 그래프(곱 경로 vs 덧셈 경로)가 사실상 중복되므로 보류했다.
- **LLM(11부)**: 스케일링 법칙(`L(N)≈(N_c/N)^{α_N}`)과 LoRA(`W=W_0+BA`, 파라미터 수 `r(d+k)` vs `d×k`)는 모두 "입력값을 바꾸면 공식이 즉시 재계산되는 계산기"형 상호작용인데, 이는 `cnn-conv-lab`의 3번(FC vs 합성곱 파라미터 수 계산기)·4번(1×1 합성곱 연산량 계산기) 기능과 상호작용 패턴이 매우 유사하다(둘 다 "두 방식의 파라미터/연산량을 나란히 비교하는 막대그래프"). 개념적 참신함은 있으나 인터랙션 설계의 참신함이 떨어진다고 판단해 보류했다.
- **베이즈학습(2부)**: 위 두 후보와 달리 완전히 새로운 수학적 대상(확률분포, 가설공간에 대한 사후확률)을 다루고, "데이터가 쌓일수록 사후확률이 좁아진다"는 서사는 이 교재의 도입부(2부)에서 등장해 3부의 정규화(릿지=MAP)를 설명하는 데도 재사용되는 근본적인 개념인데도 지금까지 어떤 앱에서도 다루지 않았다. 상호작용도 "데이터 포인트를 하나씩 추가하며 막대그래프로 표현된 사후확률 분포가 재분배되는 것을 지켜본다"는, 기존 두 앱에 없던 새로운 패턴이다. 이상의 이유로 이번 3번째 앱의 주제로 베이즈학습(2부)을 최종 선정한다.

## 3. 핵심 기능 목록 (5개 이내)

1. **베이즈 정리 계산기(기저율 오류)**: 민감도 `p(양성|암)`, 위양성률 `p(양성|암 아님)`, 사전확률 `p(암)` 슬라이더를 조절하면 전체 확률의 법칙으로 `p(양성)`을 구하고 베이즈 정리로 `p(암|양성)`을 실시간 재계산한다. 100명 아이콘 격자(진짜 양성/위양성/진짜 음성/위음성 색 구분)로 "민감도가 높아도 사후확률은 낮을 수 있다"는 기저율 오류를 시각적으로 보여준다. 교재 기본값(0.8, 0.096, 0.01 → 약 0.078)을 초깃값으로 제공.
2. **조건부확률 방향성 실습(성적표 예제)**: 2×2 분할표(열심히 공부함/안 함 × A+ 받음/그 외) 4칸의 숫자를 직접 입력하면 `p(A+|공부함)`과 `p(공부함|A+)`를 나란히 계산해 표시한다. 교재의 두 예시 표(18·2·2·8 및 10·10·2·8)를 프리셋 버튼으로 불러올 수 있어, 두 조건부확률이 같을 수도 다를 수도 있음을 직접 대조한다.
3. **생성적 분류기(오션뷰 호텔 MAP 분류)**: 호텔 A/B/C(또는 사용자가 이름을 바꾼 클래스) 각각의 우도 `p(x|y=c)`와 사전확률 `p(y=c)` 슬라이더를 조절하면, 베이즈 정리로 정규화된 사후확률 막대그래프가 즉시 갱신되고 가장 높은 막대(MAP 선택)가 강조 표시된다. 교재 기본값(0.75/0.25/0.50, 균등 사전확률)을 초깃값으로 제공.
4. **숫자게임(Number Game) 가설공간 탐색기**: 1~100 격자에서 숫자를 클릭해 데이터 `D`에 추가/제거하면, 고정된 가설공간(짝수 전체·홀수 전체·3의 배수·2의 거듭제곱·10 이하의 수 등 교재에 준하는 10여 개 규칙) 중 `D`와 모순되지 않는 가설들의 우도 `p(D|h)=(1/|h|)^N`을 계산해 막대그래프(로그 스케일)로 비교한다. `D={16}`과 `D={16,8,2,64}`를 프리셋 버튼으로 재현해 데이터가 늘수록 우도비가 벌어지는 것(교재 예시: 약 5000:1)을 확인시킨다.
5. **MLE ↔ MAP 갈림길 토글**: 4번 기능에 "사전확률 반영" on/off 토글과 가설별 사전확률(자연스러움 가중치) 슬라이더를 추가한다. off(MLE, 균등사전) 상태와 on(MAP) 상태에서 사후확률 순위가 달라질 수 있음을 나란히 비교하고, 데이터 개수 `N`을 늘려가는 슬라이더로 `log p(D|h)`(N에 비례해 커짐)와 `log p(h)`(N과 무관한 상수)를 누적 막대로 겹쳐 그려, `N`이 커질수록 사전확률 항의 영향력이 우도 항에 압도되어 MAP가 MLE로 수렴하는 과정(교재 443~462행)을 시각화한다.

## 5. 주요 UI 구성

- **공통 레이아웃**: 세로로 스크롤하는 5개 섹션(핵심 기능 1~5에 대응), 상단에 섹션 이동용 앵커 네비게이션 바(모바일에서는 드롭다운 또는 가로 스크롤 탭으로 축소) — `cnn-conv-lab`과 동일한 톤 유지.
- **섹션 1 (기저율 오류 계산기)**: 좌측에 민감도/위양성률/사전확률 슬라이더 3개와 계산 과정(분수 형태의 베이즈 정리 대입식)을 텍스트로 단계별 표시, 우측에 100명 아이콘 격자(4가지 색: 진짜양성/위양성/진짜음성/위음성)와 결과 `p(암|양성)` 큰 글씨 표시.
- **섹션 2 (조건부확률 방향성)**: 2×2 입력 표(숫자 직접 수정 가능) + 교재 예시 2개를 불러오는 버튼 + `p(A+|공부함)`과 `p(공부함|A+)` 두 값을 나란히 큰 글씨로 대조.
- **섹션 3 (생성적 분류기)**: 클래스 개수(기본 3개, 이름/우도/사전확률 편집 가능)를 위한 입력 폼 + 사후확률 Chart.js 막대그래프(MAP로 선택된 막대는 색상 강조) + 정규화 상수 `p(x)` 계산 과정 텍스트.
- **섹션 4 (숫자게임 가설공간 탐색기)**: 좌측 1~100 숫자 격자(클릭으로 데이터 추가/제거, 선택된 숫자 강조), 우측 가설 목록별 `p(D|h)` 로그 스케일 막대그래프(모순되는 가설은 회색으로 표시), 하단에 데이터셋 프리셋 버튼(`{16}`, `{16,8,2,64}` 등).
- **섹션 5 (MLE ↔ MAP 갈림길)**: 4번 섹션 결과를 재사용하는 토글 스위치(사전확률 반영 on/off) + 가설별 사전확률 슬라이더 + 데이터 개수 `N` 슬라이더(합성 데이터 자동 생성) + `log p(D|h)`와 `log p(h)` 두 항을 누적 막대로 겹쳐 그리는 Chart.js 그래프, `N`이 커질수록 사전확률 항의 비중이 시각적으로 작아짐을 표시.
- **모바일 대응**: 각 섹션의 좌/우 배치는 좁은 화면에서 세로로 스택.

## 6. 사용할 기술

- **기본**: 순수 HTML/CSS/JavaScript. 100명 아이콘 격자와 1~100 숫자 격자는 DOM(div/span) 또는 canvas 2D API 중 구현이 더 단순한 쪽으로 직접 렌더링(별도 시각화 라이브러리 불필요).
- **차트**: `apps/cnn-conv-lab/js/vendor/chart.umd.min.js`에 이미 로컬 벤더링된 Chart.js 사본을 그대로 복사해 재사용(섹션 3의 사후확률 막대그래프, 섹션 4의 로그우도 막대그래프, 섹션 5의 누적 막대그래프). 신규 CDN 추가나 새 라이브러리 벤더링 불필요.
- **연산**: 모든 계산(베이즈 정리, 전체 확률의 법칙, 강한 표집 가정 우도 `(1/|h|)^N`, 로그우도/로그사전확률 합산)이 스칼라 연산으로 충분하므로 별도 통계/선형대수 라이브러리 불필요.

## 7. 구현 범위 외

- 연속확률변수의 켤레사전분포(가우시안-가우시안, 베타-베르누이 등) 기반 사후분포 시각화 — 2부 원문은 이산 가설공간(숫자게임)과 이산 사건(암 검사, 호텔) 예제만 다루므로, 연속분포 밀도함수 애니메이션은 이번 버전에서 다루지 않는다(추후 별도 앱 후보로 남겨둘 수 있음).
- 실제 EM 알고리즘이나 가우시안 혼합모델(5부) — 과거 기각 사유(웹에 유사 사례 많음)가 유효하므로 여전히 범위 밖.
- 붓꽃(iris) 데이터셋 등 실제 다차원 데이터 기반 생성적 분류(나이브 베이즈 학습) — 섹션 3은 교재의 오션뷰 호텔 예제처럼 사용자가 우도값을 직접 입력하는 방식으로 한정하고, 실제 데이터로부터 우도를 추정하는 절차(가우시안 나이브베이즈 등)는 포함하지 않는다.
- 숫자게임의 가설공간을 사용자가 자유롭게 새 규칙(임의 정규식/조건)으로 추가하는 기능 — 이번 버전은 교재에 준하는 고정된 가설 목록(10여 개)만 제공하고, 가설공간 자체를 편집하는 UI는 제외한다.
- 사후확률 계산에 몬테카를로 근사나 표본추출(MCMC 등)을 사용하는 기능 — 모든 가설공간이 유한하고 작아 닫힌 형태로 정확히 계산 가능하므로 근사 기법은 필요하지 않으며 다루지 않는다.
