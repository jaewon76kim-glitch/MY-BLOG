## 1. 앱 이름 및 한 줄 설명

**폴더명**: `apps/gradient-descent-lab/`

**한 줄 설명**: 선형회귀의 손실함수(MSE) 등고선 위에서 경사하강법이 실제로 어떻게 움직이는지 — 학습률, 배치/확률적 방식, 릿지 정규화(MAP)의 효과까지 — 직접 조작하며 눈으로 확인하는 시뮬레이터.

## 2. 왜 이 주제인가 (선정 이유)

교재 3부(`회귀: 데이터에 곡선을 맞추는 방법`)는 이 책 전체에서 가장 먼저 등장하는, 그리고 이후 모든 장(4부 로지스틱회귀, 7부 역전파, 11부 RLHF의 정책 최적화까지)에서 반복 재사용되는 핵심 도구인 **경사하강법**을 유도한다. 교재가 직접 다루는 개념들 —

- `MSE(w)`가 `w`에 대한 이차함수(포물면)이므로 **볼록함수**이고, 그래서 어디서 출발해도 전역 최적해로 수렴한다 (3부 §2.3)
- 학습률 `η`이 너무 작으면 느리고, 너무 크면 발산/진동한다 (3부 §2.2)
- 배치 경사하강법 vs 확률적 경사하강법(SGD)의 경로 차이 (3부 §2.4)
- 정규방정식(닫힌 형태 해)과 경사하강법(수치적 해)이 같은 지점에 수렴한다 (3부 §2, §5)
- 릿지회귀(MAP)는 가우시안 사전확률을 준 것이며, `RSS(w) + λ‖w‖²`를 최소화하므로 등고선 자체가 원점 쪽으로 당겨진다 (3부 §6)

— 는 모두 수식만으로는 직관을 얻기 어렵고, **등고선 위에서 점이 실제로 이동하는 모습**을 보여줄 때 가장 효과적으로 전달된다. 또한 이 개념은 SVM(6부)을 제외한 사실상 모든 후속 장(로지스틱회귀, 퍼셉트론, 역전파, RLHF)에서 재사용되는 "이 책 전체를 관통하는" 도구이므로, 첫 ML 웹앱 주제로서 대표성이 가장 크다.

다른 후보(SVM 커널트릭 결정경계, k-평균/GMM 애니메이션, 어텐션 가중치 시각화)도 검토했으나:
- SVM 커널트릭은 쌍대문제 QP를 실시간으로 풀어야 해 구현 난이도가 높고,
- k-평균/GMM 애니메이션은 이미 웹에 유사 사례가 많으며,
- 어텐션 시각화는 실제 언어 데이터나 학습된 임베딩 없이는 장난감 예제에 그쳐 교육적 설득력이 떨어진다.

이번 주제는 순수 수학(닫힌 형태 수식)만으로 정확하게 계산 가능해 구현 리스크가 낮으면서도, 교재가 가장 공들여 유도하는 개념을 정면으로 다룬다는 점에서 선택했다.

## 3. 핵심 기능 목록 (5개 이내)

1. **데이터 점 편집**: 산점도 캔버스를 클릭해 `(x, y)` 데이터 점을 추가/삭제한다. 점이 바뀔 때마다 `MSE(w₀, w₁)` 등고선(2D contour, `w₀`=절편, `w₁`=기울기 평면)을 즉시 재계산해 다시 그린다.
2. **경사하강법 애니메이션**: 등고선 패널 위 한 점을 클릭해 시작 `(w₀, w₁)`을 지정하고, 학습률 `η` 슬라이더를 조절하면 `w_next = w - η∇MSE(w)` 규칙에 따라 이동 경로를 등고선 위에 애니메이션으로 그린다. `η`를 과도하게 크게 하면 경로가 발산/진동하는 것도 그대로 보여준다(교재 3부 §2.2의 직접 검증).
3. **배치 vs 확률적(SGD) 비교 토글**: 같은 시작점·학습률에서 전체 데이터로 기울기를 구하는 배치 GD(매끄러운 경로)와, 매 스텝 무작위 1개 데이터로 근사하는 SGD(잡음 섞인 경로)를 나란히 오버레이해 비교한다.
4. **정규방정식 해(OLS) 마커**: `ŵ_OLS = (XᵀX)⁻¹Xᵀy`를 닫힌 형태로 계산해 등고선 최저점에 고정 마커로 표시한다. 경사하강법이 반복을 거듭할수록 이 마커에 수렴해 가는 것을 시각적으로 확인시킨다.
5. **릿지 정규화 λ 슬라이더**: `λ`를 0보다 크게 올리면 목적함수가 `RSS(w) + λ‖w‖²`로 바뀌어 등고선 형태와 최솟값 위치가 원점 쪽으로 이동하는 것을 실시간으로 보여준다(3부 §6, MAP=가우시안 사전확률의 효과). `λ=0`이면 4번의 OLS 마커와 정확히 일치함을 대조할 수 있다.

## 5. 주요 UI 구성

- **좌측 패널 — 산점도(scatter) 캔버스**: x축/y축 데이터 공간. 클릭으로 점 추가, 점 위 클릭(또는 우클릭)으로 삭제. 현재 `w₀, w₁`(경사하강법이 도달한 최신 값)에 해당하는 회귀직선을 데이터 위에 겹쳐 그린다.
- **우측 패널 — 손실 등고선(contour) 캔버스**: 가로축 `w₀`, 세로축 `w₁`, 색상/등고선으로 `MSE(w)` 값을 표현(볼록한 "그릇" 모양이 등고선으로 보임). 사용자가 클릭한 시작점에서 경사하강 경로(점들을 잇는 선, 스텝마다 애니메이션)를 그리고, OLS 해 위치에 별표 마커를 고정한다.
- **컨트롤 바(하단 또는 사이드바)**:
  - 학습률 `η` 슬라이더 (예: 0.001 ~ 1.0, 로그 스케일)
  - 배치/SGD 토글 스위치
  - 정규화 강도 `λ` 슬라이더 (0 ~ 큰 값)
  - "시작/일시정지/리셋" 버튼
  - 현재 스텝 수, 현재 MSE 값, 수렴 여부 텍스트 표시
- **하단 보조 그래프(선택)**: 반복 스텝에 따른 `MSE` 값 감소 곡선(수렴 속도를 학습률별로 비교하는 라인 차트). 구현 시 아래 6번 항목의 Chart.js 벤더 사본을 재사용할 수 있으면 이 그래프에 사용한다.

## 6. 사용할 기술

- **기본**: 순수 HTML/CSS/JavaScript, canvas 2D API로 산점도와 등고선을 직접 렌더링(외부 시각화 라이브러리 없이 구현 가능한 규모).
- **선택적 보조 그래프**: 반복별 MSE 감소 곡선이 필요할 경우, `apps/link-budget-calc/js/vendor/chart.umd.min.js`에 이미 로컬 벤더링된 Chart.js 사본이 있으므로 이를 그대로 복사해 재사용한다(신규 CDN 추가나 새 라이브러리 벤더링 불필요). 등고선/경사하강 경로 자체는 Chart.js로 그리기 부적합하므로 canvas 직접 구현을 유지한다.
- **행렬 연산**: 2×2 규모(`w₀, w₁`만 다루는 단순 선형회귀)이므로 별도 선형대수 라이브러리 없이 직접 스칼라/2차원 벡터 연산으로 충분하다.

## 7. 구현 범위 외

- 다항회귀(고차항)나 다변량(3차원 이상 `w`) 확장 — 등고선 시각화가 2차원(w₀, w₁)에 묶여 있으므로 이번 버전에서는 단순선형회귀(특징 1개)로 한정한다.
- 로지스틱회귀 NLL 등고선(볼록성은 같지만 다른 챕터 주제이므로 제외; 추후 별도 앱 후보로 남겨둔다).
- 라플라스 잡음 가정에 따른 MAE(로버스트 회귀) 손실 비교.
- 미니배치 크기 조절(2 이상) — 배치 vs 순수 SGD(크기 1) 두 가지 모드만 제공하고, 임의 크기의 미니배치 슬라이더는 포함하지 않는다.
- 데이터 저장/불러오기(localStorage, 파일 업로드/다운로드) 기능.
- 3D(진짜 입체) 손실 표면 렌더링 — 2D 등고선(색상 맵 + 컨투어 라인)으로 대체하며, WebGL 등 3D 렌더링은 사용하지 않는다.
