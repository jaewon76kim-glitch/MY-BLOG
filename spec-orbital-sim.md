# spec-orbital-sim — 케플러 궤도역학 시각화

## 0. 소스 확인 메모 (작성 전 확인한 사실)

`/mnt/c/Users/hugok/Claude/Projects/doc/위성통신/위성통신_소스.tex`의 8장 1절
`궤도역학 기초: 케플러 법칙과 위성 궤도주기`(689~761행)를 직접 읽고 확인했다.

- 이 절이 실제로 유도하는 것은 **원궤도(circular orbit)** 힘 균형뿐이다.
  - 힘 균형: `GM_E m / r^2 = m v^2 / r` → 궤도속도 `v = sqrt(GM_E / r)` (질량과 무관, r에만 의존)
  - 궤도주기(케플러 제3법칙): `T = 2π sqrt(r^3 / GM_E)`, 즉 `T^2 = (4π^2 / GM_E) r^3`
  - 상수: `GM_E = 3.986×10^14 m^3/s^2`, `R_E ≈ 6378 km`
  - GEO 고도 유도: 항성일 `T=86164s`를 대입 → `r_GEO ≈ 42164 km` → `h_GEO = r_GEO - R_E ≈ 35786 km`,
    이때 `v_GEO ≈ 3.07 km/s`
  - LEO/MEO/GEO 분류표(정확한 수치는 본문 인용):
    - LEO: 고도 약 300~2000km, 주기 약 90~127분 (Starlink 고도 약 550km·주기 약 95분,
      AST SpaceMobile 약 700km대, Amazon Leo 약 590~630km대)
    - MEO: 고도 약 2000~35786km (GPS 고도 약 20200km·주기 약 12시간, Galileo 등)
    - GEO: 고도 약 35786km, 주기 24시간(항성일)
- **본문에는 이심률·장반경·근지점/원지점·비스비바 방정식·케플러 제2법칙(면적속도 일정)이 명시적으로
  등장하지 않는다.** 이 장은 원궤도만 다룬다(타원은 언급 없음).
- 지침서(`plan-agent-orbital-instructions.md`)는 타원 궤도, 이심률, 비스비바 방정식, 케플러 2법칙
  시각화를 명시적으로 요구하므로, 이 앱은 본문의 원궤도 특수해(`e=0`)를 **표준 케플러 궤도역학의
  일반해(타원, `e>0`)로 자연스럽게 확장**한다. 사용하는 일반식은 교재의 원궤도 공식과 정확히
  정합적(consistent)이다 — `e=0`으로 두면 앱의 모든 공식이 교재의 원궤도 공식으로 정확히 환원된다.
  - 케플러 제3법칙(장반경 `a` 사용): `T^2 = (4π^2/GM_E) a^3` — 교재의 `r`을 `a`로 일반화한 것뿐.
  - 비스비바 방정식: `v(r) = sqrt(GM_E (2/r - 1/a))` — `a=r`(원궤도)일 때 교재의 `v=sqrt(GM_E/r)`와 일치.
  - 근지점/원지점 반지름: `r_p = a(1-e)`, `r_a = a(1+e)`; 근지점/원지점 속도는 비스비바에 대입.
  - 케플러 제2법칙(면적속도 일정): 근지점 부근에서 빠르고 원지점 부근에서 느림 — 표준 결과이며
    별도 계산 없이 각속도가 거리 제곱에 반비례(`r^2 dθ/dt = const`)함을 애니메이션으로 보여준다.
  - 이 확장 사실을 앱 하단에 "본문은 원궤도 특수해를 다루며, 이 앱은 이를 일반 타원 궤도로 확장한
    것" 이라고 한 줄로 명시한다(교재 내용과 앱의 확장 범위를 사용자에게 투명하게 알림).

## 1. 앱 이름 및 한 줄 설명

**폴더명**: `apps/kepler-orbit-sim`

**한 줄 설명**: 장반경·이심률을 조절해 타원 궤도의 모양과 위성의 케플러 2법칙 운동(근지점에서
빠르고 원지점에서 느림)을 실시간으로 시각화하고, LEO/MEO/GEO 궤도주기·속도를 케플러 3법칙과
비스비바 방정식으로 비교하는 궤도역학 시뮬레이터.

`doppler-sim`(위성 한 번의 pass 동안 도플러·RTT 변화, 원궤도 가정, 지상국 관측 시점의 앙각/거리
변화)과 겹치지 않도록, 이 앱은 지상국 관측 시점이 아니라 **궤도 자체의 기하학**(타원 형태, 초점에
있는 지구, 근지점/원지점, 면적속도 일정)에 집중한다.

## 2. 핵심 기능 목록

1. **타원 궤도 실시간 애니메이션** — 장반경(a)·이심률(e) 슬라이더로 궤도 모양이 즉시 바뀌고,
   위성이 케플러 제2법칙에 따라 근지점 부근에서 빠르게, 원지점 부근에서 느리게 움직인다(등시간
   간격으로 쓸고 지나가는 부채꼴 면적이 항상 같음을 부채꼴 하이라이트로 시각화).
2. **LEO/MEO/GEO 프리셋 비교** — 버튼 클릭 시 해당 궤도(원궤도 기본, LEO는 이심률 슬라이더로
   타원 예시도 선택 가능)로 전환되며 고도·궤도주기·궤도속도가 케플러 제3법칙/비스비바 방정식으로
   자동 계산·표시된다.
3. **실시간 수식 패널** — 케플러 제3법칙(`T^2=4π²a³/GM_E`)으로 구한 궤도주기와, 비스비바
   방정식(`v=√(GM_E(2/r-1/a))`)으로 구한 위성의 현재 순간속도·근지점속도·원지점속도를 텍스트로
   동시에 표시한다.
4. **근지점/원지점 마커 및 궤도 파라미터 표시** — 궤도 그림 위에 근지점(perigee)·원지점(apogee)
   위치와 거리(고도)를 표시하고, 이심률 e, 장반경 a, 근지점고도, 원지점고도를 숫자로 병기한다.
5. **(확장, 선택) 지상궤적(ground track) 뷰** — 궤도경사각 슬라이더를 추가해 세계지도(단순
   equirectangular 투영) 위에 위성이 지나가는 지상궤적을 그린다. 시간 예산이 부족하면 Build 단계에서
   제외 가능(선택 사항으로 명시).

## 4. 주요 UI 구성

- **상단 헤더**: 앱 제목 + 한 줄 설명 (기존 앱들과 톤 통일).
- **좌측 입력 패널** (`aside`, 기존 앱들과 동일한 `param-group`/`preset-btn`/`slider` 클래스 재사용):
  - 장반경 `a` 슬라이더 (예: 7,000km~45,000km)
  - 이심률 `e` 슬라이더 (0.0~0.8 정도, 0=원궤도)
  - LEO / MEO / GEO 프리셋 버튼 (클릭 시 a, e 자동 세팅)
  - (선택) 궤도경사각 `i` 슬라이더 — 지상궤적 뷰용
  - 재생/일시정지, 배속(×1/×5/×20 등) 컨트롤 — doppler-sim과 동일한 패턴
- **중앙 출력 패널**:
  - 메인 canvas: 지구를 초점에 둔 타원 궤도, 위성이 애니메이션으로 이동, 근지점/원지점 마커,
    현재까지 쓸고 지나간 면적속도 부채꼴을 반투명 색으로 하이라이트
  - (선택) 하단 또는 탭 전환으로 지상궤적 세계지도 뷰
- **우측/하단 수치 패널**:
  - 궤도주기 T (분/시간 단위 자동 전환)
  - 근지점속도 `v_p`, 원지점속도 `v_a`, 현재 순간속도 `v(t)`
  - 근지점고도, 원지점고도, 이심률, 장반경 숫자 표시
  - 현재 선택된 프리셋명(LEO/MEO/GEO/커스텀) 배지
- **하단 안내 문구**: "본문(8장 1절)은 원궤도 특수해를 다루며, 이 앱은 이를 일반 타원 궤도로
  확장했습니다" 한 줄 각주.

## 5. 사용할 기술

- 순수 HTML/CSS/JS, canvas 2D API로 궤도·위성 렌더링 (doppler-sim의 `renderer.js` canvas 패턴과
  동일한 방식 재사용 — `canvas.getContext('2d')`, `requestAnimationFrame` 루프).
- 케플러 방정식(평균근점이각 → 이심근점이각, Newton-Raphson 수치해)으로 시간에 따른 위성 위치를
  계산 — 별도 라이브러리 불필요, 순수 JS로 충분(수십 줄 수준).
- 수치 표시용 Chart.js는 필수는 아니지만, "궤도 위치별 순간속도" 같은 보조 그래프를 넣고 싶다면
  기존 3개 앱처럼 `js/vendor/chart.umd.min.js`를 이 앱 폴더에도 **복사**해 자체 완결성을 유지한다
  (다른 앱과 공유 참조하지 않음 — CLAUDE.md의 "모든 웹앱은 /apps/{앱이름}/ 폴더 안에 자체 완결된다"
  원칙).
- 지상궤적(선택 기능)은 외부 지도 이미지/라이브러리 없이 canvas 위에 단순화된 대륙 윤곽(SVG path
  또는 좌표 배열)을 직접 그리거나, 위경도 격자선만으로 최소 구현한다 — 외부 CDN 지도 라이브러리는
  쓰지 않는다(모바일 대응 및 외부 의존성 최소화 원칙).
- 모바일 대응: 기존 앱과 동일하게 좁은 화면에서 좌측 패널이 상단으로 접히는 반응형 레이아웃.

## 6. 구현 범위 외

- 3차원(3D) 궤도 시각화(경사각·승교점 적경 등 완전한 6개 궤도요소 반영)는 다루지 않는다 — 이 앱은
  궤도면 내부의 2D 타원 기하(장반경, 이심률, 근지점/원지점, 면적속도)에 집중한다. 지상궤적 기능만
  예외적으로 궤도경사각 1개 요소를 추가로 쓴다.
- 섭동(J2 편평도 효과, 대기 항력에 의한 궤도 감쇠 등) 미포함 — 순수 2체 문제(two-body problem)
  케플러 궤도만 다룬다.
- 실제 위성 TLE(Two-Line Element) 데이터 연동이나 실시간 위성 위치 조회는 하지 않는다 — 프리셋은
  대표값(교재에 인용된 Starlink/GPS/GEO 고도)을 하드코딩한 예시로만 제공한다.
- 도플러 천이·RTT·링크버짓·변조 성능 계산은 다루지 않는다(이미 `doppler-sim`,
  `link-budget-calc`, `constellation-sim`이 다룸) — 이 앱은 궤도 기하학과 케플러 법칙에만 집중한다.
- 위성 간 통신(inter-satellite link)이나 군집(constellation) 커버리지 분석은 다루지 않는다.
