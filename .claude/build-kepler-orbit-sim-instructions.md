## 물리/수학 (orbit.js)

### 상수

```
GM_E = 3.986e14   m^3/s^2
R_E  = 6378 km    (지구 반경, 표시용)
```

### 기본 공식 (교재 8장 1절의 원궤도 공식을 장반경 a를 이용한 일반 타원 공식으로 확장 — a=r일 때 교재 공식과 정확히 일치해야 함)

```
궤도주기(케플러 3법칙):   T = 2π * sqrt(a^3 / GM_E)          // a는 m 단위
비스비바 방정식:          v(r, a) = sqrt(GM_E * (2/r - 1/a))  // m/s
근지점/원지점 반지름:      r_p = a*(1-e),  r_a = a*(1+e)
근지점/원지점 속도:        v_p = v(r_p, a),  v_a = v(r_a, a)
```

**검증**: e=0이면 r_p=r_a=a=r 이고 v(r,a=r) = sqrt(GM_E/r) = 교재의 원궤도 속도 공식과 정확히 같아야 한다.
GEO(T=86164s)를 위 공식에 대입하면 a≈42164km가 나와야 한다(교재 값과 일치 확인).

### 시간에 따른 위성 위치 (케플러 방정식)

```
평균운동:      n = 2π / T
평균근점이각:   M(t) = n * t   (mod 2π)
케플러 방정식:  M = E - e*sin(E)   →  Newton-Raphson으로 E를 수치적으로 구함
               E_{k+1} = E_k - (E_k - e*sin(E_k) - M) / (1 - e*cos(E_k))
               초기값 E_0 = M, 5~10회 반복이면 충분히 수렴
반지름:        r(E) = a * (1 - e*cos(E))
진근점이각:    tan(ν/2) = sqrt((1+e)/(1-e)) * tan(E/2)   // atan2로 사분면 안전하게 처리
평면 좌표:     x = r * cos(ν),  y = r * sin(ν)   // 지구는 초점(0,0)에 위치 (근지점이 +x축 방향이 되도록 배치)
```

**케플러 2법칙(면적속도 일정) 자동 만족 확인**: `r² * dν/dt`가 시간에 관계없이 일정해야 한다(각운동량 보존).
Build 후 몇 개의 다른 시각 t에서 이 값을 계산해 상수인지(오차 1% 이내) 확인할 것 — 이게 곧 "근지점에서
빠르고 원지점에서 느림"이 올바르게 구현됐다는 증거다.

### LEO/MEO/GEO 프리셋 (모두 원궤도, e=0 기본값)

```
LEO: a = 6378+550 = 6928 km   (Starlink 예시)
MEO: a = 6378+20200 = 26578 km  (GPS 예시)
GEO: a = 42164 km
```

### 지상궤적(선택 기능, groundtrack.js)

궤도경사각 `i`, 근지점인수=0, 승교점적경=0으로 단순화:

```
위도:   φ = asin(sin(i) * sin(u))          // u = ν (인수단순화)
경도:   λ = atan2(cos(i)*sin(u), cos(u)) - ω_earth * t   // ω_earth = 2π/86164 rad/s (지구자전)
```

세계지도는 외부 이미지/라이브러리 없이 위경도 격자선만으로 최소 구현(위도 15° 간격 가로선, 경도 30° 간격
세로선 정도)해도 충분하다. 시간이 부족하면 이 파일/기능은 생략하고 완료 보고에 "지상궤적은 시간 관계상
제외"라고 명시한다(spec에서 선택 사항으로 이미 명시됨).

## 구현 규칙

1. `type="module"` 사용 금지. 모든 JS는 전역 변수로.
2. CDN 사용 금지.
3. 빌드 도구 없음.
4. 모바일 대응 필수.
5. a 또는 e 슬라이더 변경 시 애니메이션을 리셋하고 새 궤도로 다시 시작한다(진행 중이던 위상은 유지해도 되고 리셋해도 됨 — 자연스러운 쪽으로 구현).
6. 재생 중일 때만 `requestAnimationFrame` 루프 동작.

## 완료 조건 (수치 검증)

- e=0, a=42164km일 때 T가 정확히 86164초(±1초)로 계산되는가.
- e=0, a=6928km(LEO)일 때 T가 약 5560~5580초(약 92.7분) 범위로 계산되는가.
- e를 0.5 정도로 놓았을 때, 근지점 근처에서 위성이 화면상 빠르게, 원지점 근처에서 느리게 움직이는 게
  육안으로도 확인되는가(애니메이션 프레임 간격이 균등한 실시간 기준일 때).
- r²·dν/dt가 궤도 여러 지점에서 거의 일정한지(각운동량 보존, 콘솔 로그나 임시 계산으로 확인 가능).
- 브라우저에서 index.html을 열었을 때 바로 동작.
- 모바일 화면(375px)에서 레이아웃이 깨지지 않음.
