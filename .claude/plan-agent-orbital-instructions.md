# Plan 서브에이전트 지침 — 궤도역학 시각화

## 목표
`위성통신_소스.tex`의 케플러 궤도역학(궤도 분류, 궤도주기 유도 등) 이론을 기반으로 인터랙티브 웹앱을 기획하고
spec 파일을 작성한다.

## 배경

doc 프로젝트의 위성통신 교재(`/mnt/c/Users/hugok/Claude/Projects/doc/위성통신/위성통신_소스.tex`)에는
케플러 궤도역학(LEO/MEO/GEO 분류, 궤도주기 유도, 이심률/장반경 등) 파트가 있다. 기존 위성통신 웹앱 3개
(`apps/doppler-sim` 도플러+RTT, `apps/link-budget-calc` 링크버짓, `apps/constellation-sim` PSK/QAM
성상도+BER) 중 `doppler-sim`이 위성 한 번의 통과(pass)를 다루긴 하지만, 케플러 궤도 자체(타원 궤도, 이심률,
근지점/원지점, 비스비바 방정식, 궤도주기 유도, LEO/MEO/GEO 비교)를 정면으로 다루지는 않는다 — 겹치지 않게
차별화된 웹앱을 기획해야 한다.

## 작업 절차

1. `위성통신_소스.tex`에서 케플러 궤도역학 관련 절(궤도 분류, 궤도주기 유도, 케플러 법칙 등)을 실제로 읽어
   정확한 수식(비스비바 방정식, 궤도주기 공식, 이심률 정의 등)을 확인한다.
2. `apps/doppler-sim/`의 spec/코드를 간단히 훑어 이미 다룬 내용(단일 pass의 도플러/RTT 애니메이션)을
   파악하고, 이번 앱은 궤도 자체의 기하학(타원, 케플러 2법칙 면적속도 일정, LEO/MEO/GEO 비교)에 집중해
   내용이 겹치지 않게 한다.
3. 다음을 반드시 포함하는 인터랙티브 웹앱을 기획한다:
   - 궤도 장반경/이심률을 조절하면 타원 궤도 형태와 위성 위치가 실시간으로 바뀌는 애니메이션(케플러 2법칙:
     근지점 부근에서 빠르고 원지점 부근에서 느리게 움직이는 것이 시각적으로 보여야 함)
   - LEO/MEO/GEO 프리셋 비교(고도, 궤도주기, 속도 자동 계산·표시)
   - 궤도주기 공식(케플러 3법칙)과 비스비바 방정식으로 계산한 속도를 실시간 텍스트로 표시
   - (선택) 지상 궤적(ground track) 시각화 — 세계지도 위에 위성이 지나가는 경로

## 출력 요구사항

`/home/hugok/MY-BLOG/spec-orbital-sim.md` 파일을 다음 항목으로 작성한다:

1. **앱 이름(폴더명, `apps/` 아래)과 한 줄 설명**
2. **핵심 기능 목록** (5개 이내)
3. **파일 구조** — `/apps/{앱이름}/` 하위 파일 목록
4. **주요 UI 구성**
5. **사용할 기술** — 순수 HTML/CSS/JS, 벤더링할 라이브러리(필요 시 기존 앱들의 `js/vendor/` 재사용 가능한지 확인)
6. **구현 범위 외**

`data-category`는 `위성통신`으로 고정. spec 파일 작성 후 완료를 보고한다. 구현은 하지 않는다.
