# Review 지침: PSK/QAM 성상도 & BER 시뮬레이터

이 파일은 Review 서브에이전트 전용 작업 지침이다. Build 서브에이전트가 이미
`/home/hugok/MY-BLOG/apps/constellation-sim/`에 앱을 만들었다. 이 결과물을 검증하고,
문제가 있으면 그 폴더 안 파일만 직접 고친다.

## 범위

**수정 가능**: `/home/hugok/MY-BLOG/apps/constellation-sim/` 폴더 안의 파일만.
**절대 건드리지 않음**: 블로그의 다른 파일 일체(index.html, css/, js/home.js, posts/, 다른 apps/* 등).

## 확인할 것

1. **브라우저 동작**: 정적 서버(`python3 -m http.server`)로 `apps/constellation-sim/index.html`을
   띄워서 실제로 콘솔 에러 없이 로드되는지, 변조방식 버튼/Eb-N0 슬라이더/샘플수 슬라이더를
   조작하면 성상도와 BER 곡선이 정상적으로 갱신되는지 확인한다.
2. **코드 문제**: `node --check`로 JS 문법, HTML의 DOM id 참조가 실제 요소와 일치하는지,
   모바일 반응형(작은 뷰포트에서 레이아웃 깨짐 없는지) 확인한다.
3. **가장 중요한 것 — QPSK 수식 검증**: Build 에이전트가 다음 문제를 보고했다.

   `/mnt/c/Users/hugok/Claude/Projects/doc/위성통신/위성통신_소스.tex` 546번째 줄 부근의
   QPSK 심볼오류율 "정확식"이

   ```
   P_s,QPSK = 2·Q(√(2·Es/N0))·[1 − 0.5·Q(√(2·Es/N0))]
   ```

   로 되어 있는데, 이걸 그대로 옮겨 구현하면 QPSK가 BPSK보다 같은 Eb/N0에서 BER이
   1000배 가까이 낮게 나온다. 이는 "Gray 코딩된 QPSK는 BPSK와 동일한 Eb/N0에서
   동일한 BER을 가진다"는 디지털통신의 잘 알려진 사실과 모순된다.

   **직접 유도로 검증해보면**: QPSK는 I/Q 두 축에 독립적인 BPSK를 싣는 것과 같다.
   각 축은 심볼에너지의 절반(Es/2 = Eb)을 담당하므로 비트오류율은
   `Q(√(2·Eb/N0))`로 BPSK와 정확히 같다(잘 알려진 결과). Es=2Eb이므로 Eb/N0 = Es/(2N0)이고,
   `√(2·Eb/N0) = √(Es/N0)`이다. 심볼오류율(두 비트 중 하나라도 틀릴 확률)은
   `1 − (1 − Q(√(Es/N0)))² = 2·Q(√(Es/N0))·[1 − 0.5·Q(√(Es/N0))]`가 된다.

   즉 인자가 `√(2·Es/N0)`가 아니라 `√(Es/N0)`이어야 한다. 이 결과는 같은 tex 파일
   552번째 줄 부근의 일반 M-PSK 근사식 `Ps ≈ 2Q(√(2Es/N0)·sin(π/M))`에 M=4를
   대입한 결과(`sin(π/4)=√2/2`이므로 `√(2Es/N0)·(√2/2) = √(Es/N0)`)와도 정확히
   일치한다 — 즉 같은 tex 파일 안에서 정확식과 근사식이 서로 모순되고, 근사식·
   표준 결과 쪽이 맞다.

   **할 일**:
   - `js/modulation.js`에서 QPSK 이론 SER/BER 계산 시 인자를 `√(Es/N0)`로 고친다
     (`√(2·Es/N0)`가 아니라).
   - 몬테카를로 실측치와 비교해서 고친 공식이 실측과 잘 맞는지, 그리고 QPSK와
     BPSK의 BER 곡선이 Eb/N0 기준으로 거의 겹치는지 수치로 재확인한다.
   - 이 발견 사항(정오표)을 review-constellation-sim.md(아래 "보고" 항목)에
     반드시 기록한다 — tex 원본 소스 자체의 잠재적 오류이므로, 앱 수정 사실과
     tex 원문 546절의 수식이 의심된다는 점을 명확히 남긴다. tex 파일 자체는
     이 서브에이전트가 고치지 않는다(범위 밖).

4. 나머지 SER/BER 공식(BPSK, M-PSK 근사, M-QAM)도 같은 방식으로 한번 더
   교차검증한다 — 특히 M-QAM의 `√M`-PAM 유도 공식이 Es/N0 정의와 일관되는지.
