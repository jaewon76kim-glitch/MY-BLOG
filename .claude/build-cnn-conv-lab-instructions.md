## 수학 (conv-math.js에 순수 함수로 구현, 다른 모듈에서 재사용)

### 1) 합성곱 슬라이딩 (섹션 1)

- 입력: `image`(n×n 픽셀값 0~255 2차원 배열), `kernel`(3×3 계수 배열)
- 한 위치의 dot product: `output[i][j] = Σ_{u,v} image[i+u][j+v] * kernel[u][v]` (패딩 없음, 스트라이드 1 기준의 기본 데모)
- 프리셋 커널(교재 관례에 맞춰 자유롭게 계수 선택 가능):
  - 엣지 검출: 예) `[[-1,-1,-1],[-1,8,-1],[-1,-1,-1]]`
  - 블러: `[[1,1,1],[1,1,1],[1,1,1]]` 전체를 9로 나눈 평균
  - 샤픈: `[[0,-1,0],[-1,5,-1],[0,-1,0]]`
  - 항등: 중앙만 1, 나머지 0
  - 커스텀: 사용자가 9칸 직접 입력
- 결과값은 시각화를 위해 정규화(예: 0~255 클리핑 또는 min-max 스케일링)해도 되나, 계산식 표시란에는
  **정규화 전 raw dot product 값**을 그대로 보여줄 것(교재의 실제 계산과 대조 가능해야 함).

### 2) 출력 크기 공식 (섹션 2)

```
n' = floor((n + 2*p - h) / s) + 1
```
- `n`=입력 크기, `p`=패딩, `h`=커널 크기, `s`=스트라이드
- 초기값: n=11, h=3, s=1, p=0 → n'=9 (spec 예시와 일치해야 함, 직접 검산: (11+0-3)/1+1=9)
- 최대풀링: 각 풀링 윈도우 내 최댓값. 평균풀링: 윈도우 내 평균값. 풀링에도 동일한 출력 크기 공식 적용
  (일반적으로 풀링은 패딩 없이 스트라이드=커널크기로 다운샘플링하는 것이 관례이나, 슬라이더로 조절 가능하게 구현).

### 3) 파라미터 수 비교 (섹션 3)

- 완전연결(FC): 입력 픽셀 수(H×W×C_in) × 출력 유닛 수(임의로 지정 가능한 채널 수×특징맵 크기, 또는
  단순화해 "입력 전체 → 출력 채널 수"로 완전연결했다고 가정) + 편향
  - LeNet-5 기본값 예시(spec 3절): 입력 32×32(1채널), 5×5 커널 6개(합성곱 1층)
    - 합성곱 파라미터: `(5*5*1 + 1) * 6 = 156`
    - 완전연결로 32×32=1024개 입력을 6개 출력 유닛에 전부 연결한다고 가정하면
      `1024 * 6 + 6 = 6150`... spec은 "약 480만개"를 언급하므로, 완전연결 비교 기준을 "입력 전체를
      출력 특징맵 전체(28×28×6=4704유닛)에 완전연결"로 잡아야 480만 근처가 나온다
      (`1024 * 4704 + 4704 ≈ 481만`). Build 시 이 가정을 명확히 정하고 UI에 어떤 가정인지 주석/설명
      텍스트로 명시할 것 (완전연결 비교는 "출력 특징맵의 각 유닛이 입력 전체와 독립적으로 완전연결된
      경우"를 뜻함).
  - 합성곱 파라미터 개수 공식: `(h*h*C_in + 1) * C_out` (편향 포함, 커널당 1개)
- 두 값을 막대그래프(Chart.js)로 비교, 로그 스케일 축 고려(자릿수 차이가 크므로).

### 4) 1×1 합성곱 연산량 절감 (섹션 4)

- 입력: `C_in`(입력 채널), `C_out`(목표 출력 채널), `k`(큰 커널 크기, 예 5×5), `C_bottleneck`(1×1로
  줄인 뒤 채널 수), 특징맵 공간 크기 `H×W`(연산량 계산에 필요, spec 기본값 재현에는 고정값 사용 가능)
- "직접 큰 커널 적용" 곱셈 횟수: `H * W * k * k * C_in * C_out`
- "1×1 병목 후 큰 커널 적용" 곱셈 횟수:
  `(1×1 병목 단계: H*W*1*1*C_in*C_bottleneck) + (큰 커널 단계: H*W*k*k*C_bottleneck*C_out)`
- spec 기본값(726만 vs 45만, 채널 120→20)을 그대로 재현할 수 있도록 `H,W,k,C_in,C_out,C_bottleneck`
  기본값을 역산해서 맞출 것(예: H=W=1 로 두고 순수 채널 곱셈 비교만 하거나, 교재 원문의 정확한 조건을
  `/mnt/c/Users/hugok/Claude/Projects/doc/머신러닝과_인공지능/머신러닝과_인공지능_소스.tex` 8부 관련
  단락에서 직접 확인해 정확히 재현할 것 — 숫자가 spec과 다르게 나오면 원문을 다시 확인해 기본값을
  맞춰라, 계산식 자체를 조작하지 말 것).

### 5) 잔차연결 기울기 흐름 (섹션 5)

- 일반(곱셈 경로만): `∂x_L/∂x_1 = Π_{i=1}^{L-1} f'_i` (각 층의 평균 기울기 인자 f'_i, 슬라이더로
  0.1~1.0 사이 공통값 하나를 모든 층에 적용하거나 층별로 동일 값 사용)
- 잔차연결 포함: `∂x_L/∂x_1 = (Π f'_i) + 1` (덧셈 지름길)
- 층 수 L을 1부터 슬라이더 최대값까지 늘려가며 두 값을 라인차트로 비교. 일반 신경망은 f'<1이면
  지수적으로 0에 수렴, 잔차연결은 항상 ≥1 유지되는 것을 시각적으로 보여줄 것. y축 로그 스케일 옵션 제공.

## 구현 규칙

1. `type="module"` 사용 금지. 모든 JS는 전역 변수로.
2. CDN 사용 금지 (Chart.js는 로컬 vendor 사본만).
3. 빌드 도구 없음.
4. 모바일 대응 필수 (900px/640px/400px 브레이크포인트, 기존 앱 관례 준수).
5. 재생 중일 때만 `requestAnimationFrame`(또는 `setInterval`) 루프 동작, 일시정지 시 정지.
6. spec 7절("구현 범위 외") — 실제 역전파 학습, 이미지 파일 업로드, 전체 아키텍처(LeNet-5 등) 층별
   시뮬레이션, TensorFlow.js 등 ML 프레임워크, 배치정규화/드롭아웃 시각화 — 구현하지 않는다.
7. 섹션 3, 4의 기본값(LeNet-5, GoogLeNet 예시)은 spec에 명시된 숫자(156 vs 480만, 726만 vs 45만)를
   실제로 재현해야 한다. 재현이 안 되면 원문 tex를 다시 확인해 가정을 맞출 것.
