## 수학 (regression.js / gradient-descent.js)

단순선형회귀: 예측 `ŷ_i = w0 + w1*x_i` (w0=절편, w1=기울기), n개 데이터 점.

```
MSE(w0,w1) = (1/n) * Σ (w0 + w1*x_i - y_i)^2
∂MSE/∂w0 = (2/n) * Σ (w0 + w1*x_i - y_i)
∂MSE/∂w1 = (2/n) * Σ (w0 + w1*x_i - y_i) * x_i

배치 GD 스텝:  w ← w - η * ∇MSE(w)          (전체 n개 데이터로 그라디언트 계산)
SGD 스텝:      w ← w - η * ∇mse_i(w)         (매 스텝 무작위 데이터 1개로 그라디언트 근사,
                                                mse_i(w) = (w0+w1*x_i - y_i)^2 의 그라디언트)

정규방정식(OLS, 닫힌 형태):
  X = [[1, x_1], [1, x_2], ..., [1, x_n]]  (n×2 행렬)
  w_OLS = (XᵀX)⁻¹ Xᵀy
  n=2 크기 행렬이므로 2×2 역행렬을 직접 스칼라 공식으로 구현 (외부 선형대수 라이브러리 불필요):
    XᵀX = [[n, Σx_i], [Σx_i, Σx_i²]]
    Xᵀy = [Σy_i, Σ(x_i*y_i)]
    2×2 역행렬: det = a*d - b*c,  inv = (1/det) * [[d,-b],[-c,a]]

릿지 정규화(λ>0):
  목적함수: RSS(w) + λ‖w‖² = Σ(w0+w1*x_i-y_i)^2 + λ*(w0²+w1²)
  그라디언트에도 2*λ*w 항이 추가됨: ∇ = [∂MSE/∂w0*n + 2λw0, ∂MSE/∂w1*n + 2λw1] (스케일 일관성 주의,
  MSE 기준으로 구현할지 RSS 기준으로 구현할지는 자유. 단 λ=0일 때 OLS 마커와 정확히 일치해야 함)
  닫힌 형태 릿지 해: w_ridge = (XᵀX + λI)⁻¹ Xᵀy  (2×2이므로 위와 동일한 방식으로 역행렬 계산)
```

**검증 기준(구현 후 직접 확인)**:
- λ=0일 때 등고선 최저점(닫힌 형태 w_ridge)이 w_OLS와 정확히 일치해야 한다.
- 학습률 η를 충분히 작게 하고 여러 스텝 반복하면 배치 GD 경로가 w_OLS에 수렴해야 한다.
- η를 과도하게 크게(예: 이 데이터 스케일에서 발산 조건 이상으로) 설정하면 경로가 발산/진동하는 것이 육안으로 보여야 한다.
- λ를 키우면 등고선 최저점이 원점(0,0) 쪽으로 이동해야 한다.

## 구현 규칙

1. `type="module"` 사용 금지. 모든 JS는 전역 변수로.
2. CDN 사용 금지.
3. 빌드 도구 없음.
4. 모바일 대응 필수.
5. 데이터 점이 바뀌면 등고선을 즉시 재계산하고, 진행 중이던 경사하강 경로는 리셋한다.
6. 재생 중일 때만 `requestAnimationFrame` 루프 동작(일시정지 시 정지).
7. spec 7절("구현 범위 외")에 명시된 기능(다항/다변량 회귀, 로지스틱 등고선, 미니배치 크기 조절,
   데이터 저장/불러오기, 3D 렌더링)은 구현하지 않는다.

## 완료 조건 (수치 검증)

- 데이터 점 3개 이상으로 OLS 해를 정규방정식으로 계산했을 때, 배치 GD를 η=0.01 정도로 충분히
  많은 스텝(수백~수천) 반복하면 w가 OLS 해에 근접(오차 1% 이내)하는가.
- λ=0일 때 릿지 닫힌 형태 해가 OLS 해와 정확히 일치하는가(부동소수점 오차 범위 내).
- λ를 늘리면 등고선 최저점이 원점 방향으로 이동하는가.
- η를 매우 크게(예: 발산 임계값 이상) 설정하면 경로가 실제로 발산/진동하는 모습을 보이는가.
- 브라우저에서 index.html을 열었을 때 바로 동작.
- 모바일 화면(375px)에서 레이아웃이 깨지지 않음.
