# 지침: 2차 독립 코드 리뷰 (Gemini 대체 리뷰)

## 배경

원래 Gemini CLI로 이 프로젝트를 리뷰시키려 했으나 Gemini CLI가 무료 API 할당량 소진으로 사용 불가능해,
대신 이 서브에이전트가 "제2의 독립적인 시선"으로 리뷰를 수행한다.

기존 `review.md`는 `apps/doppler-sim` 앱 하나에 대한 체크리스트 기반 검증 결과이며 PASS 판정을 내렸다.
이 리뷰는 그것과 **중복되지 않는, 새로 발견한 문제**만 골라내는 것이 목적이다.

## 범위

프로젝트 루트: `/home/hugok/MY-BLOG`

다음을 포함해 프로젝트 전체 코드를 리뷰한다 (doppler-sim 앱에만 국한하지 말 것):

- 블로그 코어: `index.html`, `post.html`, `css/*.css`, `js/home.js`, `js/post.js`, `js/theme.js`
- 웹앱: `apps/doppler-sim/` (단, `js/vendor/chart.umd.min.js`는 외부 라이브러리이므로 리뷰 대상에서 제외)
- 글 데이터: `posts/index.json`, `posts/*.md`
- 스킬/설정: `.claude/skills/post-article/SKILL.md`, `CLAUDE.md`

## 먼저 읽을 것

1. `/home/hugok/MY-BLOG/review.md` — 기존 doppler-sim 리뷰. 이미 다룬 항목(별 반짝임 코스메틱 이슈 등)은 다시 지적하지 말 것.
2. `/home/hugok/MY-BLOG/CLAUDE.md` — 프로젝트 규칙 (웹앱은 /apps/{앱}/ 안에 자체 완결, 외부 라이브러리 최소화, 모바일 대응 등).

## 리뷰 관점 (예시, 이 항목들에 한정하지 않음)

- **보안**: `js/post.js`가 `marked.min.js`로 마크다운을 렌더링할 때 XSS 방지(sanitize)가 되어 있는가? `innerHTML` 사용처에 사용자/외부 입력이 그대로 들어가는 곳은 없는가?
- **정확성/버그**: 로직 오류, null/undefined 참조 가능성, 이벤트 리스너 중복 등록, 메모리 누수.
- **일관성**: `CLAUDE.md`에 명시된 규칙(카테고리 9종 고정, data-category와 app-badge 일치, 외부 CDN 최소화 등)을 실제 코드가 지키고 있는가?
- **모바일 대응**: 미디어 쿼리 누락, 뷰포트 문제.
- **접근성**: alt 텍스트, 시맨틱 태그, 키보드 조작 가능 여부.
- **성능**: 불필요한 재계산, 큰 파일 크기, 렌더링 병목.
- **데이터 무결성**: `posts/index.json`과 실제 `posts/*.md` 파일 간 정합성 (파일 존재 여부, 카테고리 값이 CLAUDE.md 목록에 있는지 등).

## 절대 하지 말 것

- 이 프로젝트의 어떤 파일도 수정하지 말 것 (읽기 전용 리뷰). 코드 수정은 이 리뷰가 끝난 뒤 사용자가 검토 후 별도로 진행한다.
- `review.md`에 이미 있는 지적을 그대로 반복해서 새 문제인 것처럼 쓰지 말 것.
- 근거 없는 추측성 지적("아마도 문제가 될 수도")은 배제하고, 실제 코드 라인을 인용해 구체적으로 지적할 것.

## 출력

리뷰 결과를 `/home/hugok/MY-BLOG/gemini-review.md` 파일로 작성한다 (새로 생성).

형식:

```markdown
# 2차 독립 리뷰 (review.md 대비 추가 발견 사항)

## 요약

(전체적으로 몇 건의 새 이슈를 발견했는지, 심각도 개요)

## 발견된 문제

### [심각도: 상/중/하] 제목

- 파일:라인
- 문제 설명
- 재현/발생 조건 (구체적인 입력이나 상황)
- 수정 제안

(문제 건수만큼 반복)

## review.md와 중복 확인

review.md에 없던 새로운 관점에서 봤음을 간단히 확인.
```

발견한 문제가 하나도 없다면 그렇게 솔직히 적을 것 (억지로 문제를 만들어내지 말 것).

작업이 끝나면 무엇을 확인했고 몇 건을 발견했는지 최종 보고에 요약할 것.
