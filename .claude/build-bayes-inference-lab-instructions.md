## 수학 (bayes-math.js에 순수 함수로 구현, 다른 모듈에서 재사용)

### 1) 베이즈 정리 계산기 — 기저율 오류 (섹션 1)

```
p(양성) = p(양성|암)*p(암) + p(양성|암 아님)*(1-p(암))
p(암|양성) = p(양성|암)*p(암) / p(양성)
```

- 초깃값: `p(양성|암)=0.8`, `p(양성|암 아님)=0.096`, `p(암)=0.01`
- 검산: `p(양성) = 0.8*0.01 + 0.096*0.99 = 0.008 + 0.09504 = 0.10304`,
  `p(암|양성) = 0.008/0.10304 ≈ 0.0777` (약 7.8%) — 이 값이 정확히 나와야 한다.
- 100명 아이콘 격자: 100명 중 실제 암 환자 수 = `100*p(암)`, 그중 양성 판정(진짜양성) =
  `100*p(암)*p(양성|암)`, 암 아닌 사람 중 위양성 = `100*(1-p(암))*p(양성|암 아님)`. 소수점은
  반올림해 100개 칸에 맞춘다(진짜양성/위양성/진짜음성/위음성 4색 구분).

### 2) 조건부확률 방향성 — 성적표 예제 (섹션 2)

```
p(A+|공부함) = 공부함∩A+ / 공부함 전체
p(공부함|A+) = 공부함∩A+ / A+ 전체
```

- 프리셋 1 (18/2/2/8): `p(A+|공부함)=18/20=0.9`, `p(공부함|A+)=18/20=0.9` (우연히 같음)
- 프리셋 2 (10/10/2/8): `p(A+|공부함)=10/20=0.5`, `p(공부함|A+)=10/12≈0.833` (서로 다름)
- 표는 "열심히 공부함/공부 안 함" × "A+ 받음/그 외" 2×2, 4칸 모두 사용자가 직접 수정 가능해야 한다.

### 3) 생성적 분류기 — 오션뷰 호텔 MAP (섹션 3)

```
p(y=c|x) = p(x|y=c)*p(y=c) / Σ_c' p(x|y=c')*p(y=c')
```

- 초깃값: 호텔 A/B/C, 우도 `p(오션뷰|A)=0.75`, `p(오션뷰|B)=0.25`, `p(오션뷰|C)=0.50`, 사전확률 균등
  `p(A)=p(B)=p(C)=1/3`.
- 검산: 정규화 전 값 `0.75/3, 0.25/3, 0.50/3` → 정규화 후 `p(A|오션뷰)=0.5`, `p(B|오션뷰)≈0.167`,
  `p(C|오션뷰)≈0.333`. A가 최댓값(MAP 선택)이어야 한다.
- 클래스 개수는 spec대로 기본 3개, 이름/우도/사전확률 편집 가능하게 구현(개수 자체를 늘리는 기능은
  선택 사항, 필수는 아님).

### 4) 숫자게임 — 가설공간 우도 (섹션 4)

```
p(D|h) = (1/|h|)^N,  단 D의 모든 원소가 h에 속할 때만(모순되면 확률 0 또는 회색 처리)
```

- 숫자 범위 1~100. 가설공간 예시(spec이 "교재에 준하는 10여 개"로 명시, 아래를 기본 목록으로 사용):
  - 짝수 전체 `|h|=50`, 홀수 전체 `|h|=50`
  - 2의 거듭제곱 `{1,2,4,8,16,32,64}` `|h|=7` — **주의**: 원문은 "$1,2,4,8,16,32,64$로 정확히 6개"라고
    썼으나 실제로 나열된 원소는 7개다(1도 2^0으로 포함됨). 원문의 우도 계산(`1/6`, `(1/6)^4`)은
    `|h|=6`을 사용해 유도되었으므로, **숫자게임의 정답 재현(아래 검산)을 맞추려면 `h_two`의 정의를
    "1을 제외한 2의 거듭제곱 {2,4,8,16,32,64}", `|h_two|=6`으로 구현**한다(원문 계산과 정확히 일치시키기
    위함). 이 가정을 UI에 짧게 명시(예: 가설 이름 옆 "|h|=6" 표시)할 것.
  - 3의 배수 `{3,6,...,99}` `|h|=33`
  - 10 이하의 수 `{1,...,10}` `|h|=10`
  - 끝자리가 0인 수 `{10,20,...,100}` `|h|=10`
  - 1부터 100 사이의 모든 수 `|h|=100`
  - 소수 `|h|=25`(1~100 소수 개수)
  - 완전제곱수 `{1,4,9,...,100}` `|h|=10`
  - 그 외 자유롭게 추가해 총 10여 개를 맞춘다.
- 검산 (D={16}): `p(D|h_two)=1/6≈0.167`, `p(D|h_even)=1/50=0.02` — h_two가 우세하나 근소.
- 검산 (D={16,8,2,64}): `p(D|h_two)=(1/6)^4≈7.72×10⁻⁴`, `p(D|h_even)=(1/50)^4=1.6×10⁻⁷`,
  우도비 `≈ 4823:1`(원문 표현 "약 5000:1"에 부합). 나머지 가설(3의 배수, 10 이하 등)은 D={16,8,2,64}와
  이미 모순되므로 `p(D|h)=0` 또는 목록에서 회색/제외 처리.
- 로그 스케일 막대그래프이므로 확률 0인 가설은 그래프에서 별도 처리(예: 축 최솟값 아래로 표시하거나
  목록에서만 "모순" 표기).

### 5) MLE ↔ MAP 갈림길 (섹션 5)

- 4번 기능 재사용. 사전확률 `p(h)`를 가설별로 슬라이더(자연스러움 가중치, 예: h_two가 h'(32 제외 2의
  거듭제곱)보다 자연스럽다는 사전확률을 더 높게 주는 식)로 부여.
- `log p(D|h) = N * log(1/|h|) = -N*log(|h|)` (N=|D|), `log p(h)`는 N과 무관한 상수.
- N(합성 데이터 개수) 슬라이더를 늘리면 `log p(D|h)`의 절댓값이 선형으로 커지고 `log p(h)`는 고정이므로,
  둘을 누적 막대(stacked bar)로 겹쳐 그리면 N이 커질수록 `log p(h)` 막대의 상대적 비중이 시각적으로
  작아지는 것을 보여줄 수 있다. off(MLE, 사전 균등)와 on(MAP, 사전 비균등) 상태를 토글로 비교.
- "32를 제외한 2의 거듭제곱" `h'`(`|h'|=5`, D={16,8,2,64}에 대해 `p(D|h')=(1/5)^4=0.0016`으로
  `h_two`의 `(1/6)^4≈0.00077`보다 우도가 높음)를 이 섹션의 데모 가설로 포함시켜, "우도만으로는 h'을
  택하지만 사전확률을 반영하면 h_two가 이긴다"는 교재의 논지를 재현할 것(spec 1-1절 근거).
