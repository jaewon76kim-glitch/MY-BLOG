# Plan 서브에이전트 지침 — 머신러닝과_인공지능 인터랙티브 웹앱 (2번째)

## 목표

`머신러닝과_인공지능_소스.tex`의 내용을 기반으로 **두 번째** 인터랙티브 웹앱을 기획하고 spec 파일을 작성한다.

## 배경

doc 프로젝트의 머신러닝 교재(`/mnt/c/Users/hugok/Claude/Projects/doc/머신러닝과_인공지능/머신러닝과_인공지능_소스.tex`)는
베이즈학습(MLE/MAP) → 선형/로지스틱 회귀(경사하강법) → 군집화(k-평균/GMM/EM) → SVM(라그랑주/KKT/커널트릭) →
신경망(퍼셉트론/역전파) → CNN → RNN/LSTM → 어텐션/트랜스포머 → LLM(스케일링 법칙/RLHF/LoRA)까지 다룬다.

이미 `apps/gradient-descent-lab/`(선형회귀 MSE 등고선 + 경사하강법 + 배치/SGD + 릿지정규화)이 이 교재를
소재로 구현되어 있다. `/home/hugok/MY-BLOG/spec-ml-sim.md`를 먼저 읽어 1번째 앱이 정확히 무엇을
다뤘는지 확인하고, **겹치지 않는 새로운 주제**를 고른다.

1번째 spec(`spec-ml-sim.md`) 2절에서 검토했다가 기각한 후보들:
- SVM 커널트릭 결정경계 — 쌍대문제 QP를 실시간으로 풀어야 해 구현 난이도가 높다고 판단해 기각
- k-평균/GMM 애니메이션 — 이미 웹에 유사 사례가 많다고 판단해 기각
- 어텐션 가중치 시각화 — 실제 언어 데이터/학습된 임베딩 없이는 장난감 예제에 그친다고 판단해 기각

이번 Plan에서는 위 기각 사유가 실제로 여전히 타당한지(예: SVM은 2D 토이 데이터+SMO의 단순화 버전이면
난이도가 감당 가능할 수도 있음) 직접 판단하되, 반드시 기각해야 하는 건 아니다 — 원본을 다시 검토해
가장 교육적인 주제를 자유롭게 고른다. 신경망/역전파, CNN, RNN/LSTM, 어텐션/트랜스포머, LLM(LoRA 등)
부분도 아직 다뤄지지 않았으니 후보에 포함한다.

1. `머신러닝과_인공지능_소스.tex`를 실제로 읽어 확인한다(위 요약은 참고용 개요일 뿐, 세부 수식/개념은
   원본에서 직접 확인할 것). `.toc` 파일이 있으면 먼저 훑어도 좋다.
2. `/home/hugok/MY-BLOG/spec-ml-sim.md`를 읽어 1번째 앱과 주제/핵심 기능이 겹치지 않는지 확인한다.
3. 교재 안에서 **가장 교육적이고 인터랙티브한 웹앱을 만들 수 있는 주제 하나**를 고른다. 단순 텍스트
   요약이 아니라, 사용자가 파라미터를 조작하며 직관을 얻을 수 있는 시각화/시뮬레이터 형태를 우선한다.
4. 기존 6개 웹앱(`apps/` 아래: constellation-sim, doppler-sim, fading-channel-sim, gradient-descent-lab,
   kepler-orbit-sim, link-budget-calc)과 주제가 겹치지 않는지 최종 확인한다.

## 출력 요구사항

`/home/hugok/MY-BLOG/spec-ml-sim-2.md` 파일을 다음 항목으로 작성한다:

1. **앱 이름(폴더명, `apps/` 아래)과 한 줄 설명**
2. **왜 이 주제인가** — 1번째 앱과 겹치지 않는 이유 포함
3. **핵심 기능 목록** (5개 이내)
4. **파일 구조** — `/apps/{앱이름}/` 하위 파일 목록
5. **주요 UI 구성** — 어떤 입력/출력/그래프가 있는지
6. **사용할 기술** — 순수 HTML/CSS/JS, 벤더링할 라이브러리(예: Chart.js — 기존 관례상 CDN보다
   `js/vendor/`에 로컬 복사 선호, `apps/link-budget-calc/js/vendor/chart.umd.min.js` 등 기존 사본
   재사용 가능한지 확인)
7. **구현 범위 외** — 이번 버전에서 제외할 기능

`data-category`는 `머신러닝과_인공지능`으로 고정. spec 파일 작성 후 완료를 보고한다. 구현은 하지 않는다.
