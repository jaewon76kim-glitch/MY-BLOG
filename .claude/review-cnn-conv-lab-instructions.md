## 알려진 주장 (검증 시 참고, Build 에이전트 자체 보고 — 직접 재검증할 것)

Build 에이전트는 다음을 주장했다. **이 주장들을 그대로 믿지 말고 코드를 직접 읽고 재계산해서 검증해라**:
- `outputSize(11,3,1,0) = 9`
- `convParams(5,1,6) = 156` (LeNet-5 합성곱 파라미터)
- `fcParams(1024,4704) = 4,821,600` (≈480만, 완전연결 비교값 — "입력 32×32=1024차원을 출력 특징맵
  6×28×28=4704차원에 완전연결"이라는 가정 사용)
- `directConvOps(11,11,5,120,20) = 7,260,000` (726만, GoogLeNet "직접 큰 커널 적용")
- Build 지침 파일의 괄호 설명("채널 120→20")은 원문과 어긋나 있었고, 실제로는
  `H=W=11, C_in=120, k=5, C_out=20(5×5 커널 20개), C_bottleneck=6(1×1 커널 6개)`이 정답이라고 주장,
  이 값으로 `bottleneckConvOps(...).total = 450,120`(≈45만)이 나온다고 주장 — **이 정정 자체가 맞는지
  원문을 직접 대조해서 확인하라. Build 에이전트가 원문을 잘못 읽었을 가능성도 열어둘 것.**
- L=1~60, f'=0.5일 때 일반 신경망 곡선은 `1.7e-18`까지 수렴, 잔차연결 곡선은 항상 `≥1` 유지
- 모든 JS `node --check` 통과, `python3 -m http.server` + curl로 10개 리소스 200 확인
- 블로그 다른 파일(index.html 등)은 전혀 수정하지 않았다고 주장

### 4. 수학/로직 재검증 (Node로 conv-math.js 등을 직접 로드하거나 로직을 재구현해 계산)

이 항목이 이 리뷰에서 가장 중요하다. 아래를 직접 숫자로 재현할 것 — Build 보고서의 수치를 베끼지 말고
네가 독립적으로 계산해서 일치하는지 대조해라.

- [ ] 섹션 1: 합성곱 dot product가 `Σ image[i+u][j+v] * kernel[u][v]` 공식대로 정확히 구현되어
      있는가. 간단한 손계산 예(예: 4×4 이미지 + 항등 커널)로 직접 대조
- [ ] 섹션 2: `n' = floor((n+2p-h)/s) + 1` 공식이 코드에 정확히 구현되어 있는가. n=11,h=3,s=1,p=0→9
      외에 다른 조합(예: n=10,h=3,s=2,p=1)으로도 직접 계산해 코드 결과와 대조
      (`floor((10+2-3)/2)+1 = floor(4.5)+1 = 5`)
- [ ] 최대풀링/평균풀링이 실제로 다른 결과를 내는지(같은 입력에 대해 두 값이 다름) 확인
- [ ] 섹션 3: 합성곱 파라미터 공식 `(h*h*C_in+1)*C_out`이 코드에 정확히 있는가.
      `(5*5*1+1)*6=156` 직접 재계산
- [ ] 섹션 3: 완전연결 비교 공식과 그 가정이 원문(1546행 부근)과 실제로 부합하는지 원문을 직접
      읽고 확인. "약 480만"이 어떤 가정에서 나온 숫자인지 원문에 명시적으로 나와 있는지, 아니면
      Build가 역산한 가정인지 구분해서 review.md에 기록
- [ ] 섹션 4: 원문(1562~1575행 부근)을 직접 읽고 GoogLeNet 예시의 실제 조건
      (H, W, C_in, k, C_out, C_bottleneck 값)을 확인. Build의 "지침 파일 괄호 설명이 틀렸고 실제로는
      H=W=11, C_in=120, k=5, C_out=20, C_bottleneck=6"이라는 주장이 원문과 정확히 일치하는지 검증.
      `directConvOps`와 `bottleneckConvOps` 공식이 코드에 올바르게 구현되어 있는지, 그 값이
      726만/45만(또는 원문의 정확한 숫자)과 실제로 일치하는지 직접 계산
- [ ] 섹션 5: 일반 신경망 `Π f'_i`와 잔차연결 `Π f'_i + 1` 공식이 코드에 정확히 구현되어 있는가.
      f'<1일 때 L이 커지면 일반 신경망 값은 0에 수렴하고 잔차연결 값은 항상 ≥1인지 직접 여러 L 값에서
      재계산해 확인
