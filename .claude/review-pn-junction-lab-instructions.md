## 알려진 주장 (검증 시 참고, Build 에이전트 자체 보고 — 직접 재검증할 것)

Build 에이전트는 다음을 주장했다. **이 주장들을 그대로 믿지 말고 코드를 직접 읽고 재계산해서
검증해라**:
- `K_B_EV*300 = 0.025851` (≈26mV)
- `n_i(300) = 6.670×10⁹ cm⁻³`
- N_A=1e17, N_D=1e15, T=300K일 때 `V_bi = 0.7352V` — **지침이 명시한 "0.6~0.7V" 범위를 약 5%
  초과한다고 자체 보고했다.** 이것이 공식 구현 버그 때문인지, 아니면 원문의 "전형적으로 0.6~0.7V"라는
  진술 자체가 근사치라서 정상적인 오차 범위인지 반드시 직접 재계산해서 판정하라. 원문의 정확한 문구
  (`전형적인 실리콘 다이오드는 V_bi≈0.6~0.7V 정도`)를 다시 확인하고, 계산에 사용된 상수(E_G, N_C_300,
  N_V_300)가 지침과 정확히 일치하는지, 공식(`V_bi = (k_BT/e)ln(N_AN_D/n_i²)`)이 코드에 올바르게
  구현되어 있는지 독립적으로 확인해 결론 내려라.
- 저농도 프리셋(N_A=N_D=1e15): V_bi=0.6162V (범위 안)
- 도핑 1e14→1e19 스윕에서 V_bi 0.497V→1.092V 단조 증가, W 3586nm→16.8nm 단조 감소
- 비대칭(N_A=1e18, N_D=1e15): x_p=1.01nm, x_n=1013.35nm, `N_A*x_p = N_D*x_n = 1.013×10¹¹` 정확히 일치
- 대칭(1e17=1e17): x_p=x_n=74.33nm=W/2
- V=-1V일 때 `I/(-I0)=1.000000`(포화), V=0.5V일 때 `I/I0≈2.5×10⁸`(폭증)
- `φ(x_n)-φ(-x_p) = E_max*W/2 = V_bi-V` 정확히 일치(3개 케이스 검산했다고 주장)
- 프리셋 3개 표(대칭/비대칭/저농도)의 V_bi/W/x_p·x_n/I0 값 (Build 보고 표 전체)
- `Ec-Ei`와 `Ei-Ev`의 합이 항상 정확히 `E_G=1.12`가 되는 자기일관성 확보 주장
- `node --check` 전부 통과, HTML 태그 균형, id 양방향 참조 일치, 동적 주입 클래스까지 CSS에 정의됨
- 블로그 다른 파일(index.html 등)은 전혀 수정하지 않았다고 주장

### 4. 물리/수학 재검증 (Node로 semiconductor-math.js를 직접 로드하거나 로직을 재구현해 계산)

이 항목이 이 리뷰에서 가장 중요하다. Build 보고서의 수치를 베끼지 말고 독립적으로 계산해서 대조하라.

- [ ] `n_i(T) = sqrt(N_c(T)*N_v(T)) * exp(-E_G/(2*K_B_EV*T))` 공식이 정확히 구현되어 있는가.
      T=300K에서 재계산해 6.6×10⁹ 근처 값이 나오는지 직접 확인
- [ ] **핵심 쟁점**: N_A=1e17, N_D=1e15, T=300K에서 `V_bi` 값을 지침 공식으로 직접 손계산(또는 별도
      스크립트로 독립 계산)하여 코드 결과(0.7352V로 보고됨)와 일치하는지 확인. 일치한다면, 이 값이
      지침의 "0.6~0.7V" 예시 범위를 벗어나는 것이 공식/상수 오류가 아니라 예시 범위 자체가 근사치였기
      때문임을 원문(`전형적인 실리콘 다이오드는 V_bi≈0.6~0.7V 정도`)과 대조해 명확히 결론짓고, 이
      결론과 근거를 review.md에 상세히 기록할 것. 만약 독립 계산 결과가 코드 결과와 다르다면 실제
      버그이므로 심각도 높음으로 기록
- [ ] `W = sqrt((2*EPS_SI*(V_bi-V)/Q)*(1/N_A+1/N_D))` 공식이 정확히 구현되어 있는가. 임의의 도핑
      조합(예: N_A=5e16, N_D=2e17)으로 직접 재계산해 코드와 대조
- [ ] `N_A*x_p = N_D*x_n` (전기적 중성 조건)이 여러 도핑 조합에서 정확히 성립하는지 확인
- [ ] `E_max = Q*N_A*x_p/EPS_SI = Q*N_D*x_n/EPS_SI` 두 표현이 코드에서 실제로 같은 값을 내는지 확인
- [ ] E(x)가 x=0에서 연속인지(두 구간 공식이 x=0에서 같은 값) 직접 계산으로 확인
- [ ] φ(x)(전위) 프로파일이 `φ(x_n)-φ(-x_p) = V_bi-V`를 만족하는지 직접 재구현해 확인(부호 규약
      `E=-dφ/dx` 주의)
- [ ] 도핑 스윕(1e14~1e19)에서 V_bi 단조 증가, W 단조 감소가 실제로 성립하는지 여러 지점에서 확인
- [ ] `p_n0 = n_i²/N_D`, `n_p0 = n_i²/N_A` 공식이 정확한가
- [ ] `I0 = Q*A*(D_p*p_n0/L_p + D_n*n_p0/L_n)` 공식이 정확한가. `D_p=(k_BT/e)*MU_P`,
      `L_p=sqrt(D_p*TAU_P)` 등 아인슈타인 관계식·확산거리 공식이 올바른지 확인
- [ ] `I(V) = I0*(exp(V/(K_B_EV*T))-1)` — 단위 처리(볼트와 eV 스케일 상쇄)가 실제로 올바른지 확인.
      V=-1V일 때 I≈-I0, V=0.5V일 때 지수적 폭증이 재현되는지 직접 계산
- [ ] `Ec-Ei = k_BT*ln(N_c/n_i)`, `Ei-Ev = k_BT*ln(N_v/n_i)`의 합이 항상 정확히 E_G(1.12)와 같은지
      직접 재계산해 확인(대수적으로 항상 성립해야 하는 항등식이므로, 코드가 이 항등식을 깨지 않는지
      float 정밀도 내에서 확인)
- [ ] 프리셋 3개(대칭/비대칭/저농도)를 코드에서 실제로 클릭했을 때(또는 해당 파라미터로 직접
      재계산했을 때) Build가 보고한 표의 숫자와 일치하는지 확인
