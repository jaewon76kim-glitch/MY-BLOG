# Plan 서브에이전트 지침 (머신러닝과_인공지능 카테고리 3번째 앱)

사용자가 이번에는 명시적으로 머신러닝 주제를 요청했다.

## 이미 구현된 앱 (겹치지 말 것)
- `apps/gradient-descent-lab/` — 3부 회귀·경사하강법. 손실함수 등고선 위에서 경사하강 경로가 수렴하는 애니메이션.
- `apps/cnn-conv-lab/` — 8부 CNN. 순전파 연산(합성곱/스트라이드/패딩/풀링/1x1conv/잔차연결)을 파라미터 조작으로 탐색.

## 과거에 기각되었던 후보와 사유 (재검토는 가능하나, 기각 사유를 반드시 원문 확인 후 반박하거나 수긍할 것)
- **SVM 커널트릭 결정경계**: 쌍대문제 QP를 실시간으로 풀어야 해 구현 난이도/수렴 안정성 리스크가 높다고 판단해 2회 연속 보류됨.
- **k-평균/GMM 애니메이션**: 웹에 유사 사례가 이미 많다는 이유로 보류됨.
- **어텐션 가중치 시각화(토이 예제)**: 실제 언어 데이터/학습된 임베딩 없이는 "소프트맥스 스케일링의 수학적 시연"에 그쳐 교재가 강조하는 서사(병목 해소·의미적 관련성)를 전달하기 어렵다고 판단해 보류됨.
- **퍼셉트론/역전파 MLP+XOR 플레이그라운드(7부)**: "파라미터를 바꾸며 최적화가 수렴하는 것을 지켜본다"는 상호작용 패턴이 gradient-descent-lab과 겹친다고 판단해 보류됨.

## 아직 검토되지 않은 유력 후보
- **베이즈학습(MLE/MAP, 2부)**: 동전 던지기/정규분포 등 단순 예제에서 데이터가 쌓일수록 사전분포가 우도에 의해 업데이트되어 사후분포가 좁아지는 과정, MLE와 MAP 추정치의 차이를 슬라이더로 탐색.
- **RNN/LSTM(9부)**: 시퀀스 처리, 기울기 소실/폭발, LSTM의 게이트(forget/input/output) 메커니즘.
- **LLM(11부, 스케일링 법칙/RLHF/LoRA)**: 스케일링 법칙 곡선, LoRA의 저랭크 분해로 파라미터 수 절감 효과 등.
- 그 외 원문을 직접 읽고 판단한 다른 주제도 좋다(단, 위 "이미 구현된 앱"과 상호작용 패턴이 겹치지 않아야 한다).

## 작업 절차
1. 위 tex 원문을 실제로 읽어(전체 정독 또는 최소 목차+후보 챕터 상세 확인) 검토한다.
2. 이미 구현된 2개 앱, 과거 기각 후보의 사유를 참고하여 **가장 교육적이고, 닫힌 형태 수식/순전파 계산만으로
   구현 리스크 낮게 만들 수 있는** 새 주제를 하나 선택한다. 학습(iteration) 애니메이션이 필요한 주제라면
   gradient-descent-lab과 상호작용 패턴이 겹치지 않도록 설계에 유의한다.
3. 선택한 주제로 구체적인 웹앱 기능을 설계한다.

## 출력 요구사항
`/home/hugok/MY-BLOG/spec-ml-sim-3.md` 파일을 다음 항목으로 작성한다:

1. **선정 이유** — 원문 근거, 기존 2개 앱과 겹치지 않는 이유, 과거 기각 후보 재검토 결과(선택하지 않은 경우 왜 여전히 기각인지, 이번 주제를 선택한 경우 왜 이것이 더 나은지)
2. **앱 이름 및 한 줄 설명** — `data-category`는 `머신러닝과_인공지능`, 폴더명은 `apps/{앱이름}/`
3. **핵심 기능 목록** (5개 이내)
4. **파일 구조** — /apps/{앱이름}/ 하위 파일 목록
5. **주요 UI 구성** — 어떤 입력/출력/그래프가 있는지
6. **사용할 기술** — 순수 HTML/CSS/JS. Chart.js가 필요하면 CDN이 아니라 기존 앱(`apps/gradient-descent-lab/js/vendor/`
   또는 `apps/cnn-conv-lab/js/vendor/`)에 이미 로컬 벤더링된 `chart.umd.min.js` 재사용 가능한지 확인한다.
7. **구현 범위 외** — 이번 버전에서 제외할 기능

spec 파일 작성 후 완료를 보고한다. **구현은 하지 않는다.** index.html, css/, js/, apps/ 하위 기존 폴더 등
다른 파일은 건드리지 않는다.
