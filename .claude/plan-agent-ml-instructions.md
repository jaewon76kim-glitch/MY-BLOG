# Plan 서브에이전트 지침 — 머신러닝과_인공지능 인터랙티브 웹앱

## 목표
`머신러닝과_인공지능_소스.tex`의 내용을 기반으로 인터랙티브 웹앱 하나를 기획하고 spec 파일을 작성한다.

## 배경

doc 프로젝트의 머신러닝 교재(`/mnt/c/Users/hugok/Claude/Projects/doc/머신러닝과_인공지능/머신러닝과_인공지능_소스.tex`)는
베이즈학습(MLE/MAP) → 선형/로지스틱 회귀(경사하강법) → 군집화(k-평균/GMM/EM) → SVM(라그랑주/KKT/커널트릭) →
신경망(퍼셉트론/역전파) → CNN → RNN/LSTM → 어텐션/트랜스포머 → LLM(스케일링 법칙/RLHF/LoRA)까지 다룬다.
이 교재는 아직 블로그 웹앱으로 다뤄진 적이 없다(기존 5개 앱은 모두 위성통신 카테고리).

## 작업 절차

1. `머신러닝과_인공지능_소스.tex`를 실제로 읽어 확인한다(위 요약은 참고용 개요일 뿐, 세부 수식/개념은 원본에서
   직접 확인할 것). 필요시 `.toc` 파일로 전체 목차를 먼저 훑어도 좋다.
2. 교재 안에서 **가장 교육적이고 인터랙티브한 웹앱을 만들 수 있는 주제 하나**를 고른다. 단순 텍스트 요약이
   아니라, 사용자가 파라미터를 조작하며 직관을 얻을 수 있는 시각화/시뮬레이터 형태를 우선한다
   (예: 경사하강법 손실 지형 시각화, k-평균/GMM 군집화 애니메이션, SVM 커널트릭 결정경계 시각화, 퍼셉트론/
   역전파 학습 과정 시각화, 어텐션 가중치 시각화 등 — 예시일 뿐 원본 확인 후 직접 판단).
3. 기존 5개 웹앱(`apps/` 아래, 모두 위성통신)과 주제가 겹치지 않는지 확인한다(머신러닝 주제이므로 자연히
   겹치지 않을 것).

## 출력 요구사항

`/home/hugok/MY-BLOG/spec-ml-sim.md` 파일을 다음 항목으로 작성한다:

1. **앱 이름(폴더명, `apps/` 아래)과 한 줄 설명**
2. **핵심 기능 목록** (5개 이내)
3. **파일 구조** — `/apps/{앱이름}/` 하위 파일 목록
4. **주요 UI 구성** — 어떤 입력/출력/그래프가 있는지
5. **사용할 기술** — 순수 HTML/CSS/JS, 벤더링할 라이브러리(예: Chart.js — 기존 관례상 CDN보다 `js/vendor/`에
   로컬 복사 선호, 필요시 `apps/link-budget-calc/js/vendor/` 등 기존 사본 재사용 가능한지 확인)
6. **구현 범위 외** — 이번 버전에서 제외할 기능

`data-category`는 `머신러닝과_인공지능`으로 고정. spec 파일 작성 후 완료를 보고한다. 구현은 하지 않는다.
