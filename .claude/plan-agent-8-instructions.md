# Plan 서브에이전트 지침

## 목표
doc 프로젝트의 PDF/TeX 교재 중 하나를 골라 인터랙티브 웹앱 하나를 기획하고 spec 파일을 작성한다.

## 중요: 카테고리 선택 제약

기존 웹앱은 총 8개다: 위성통신 카테고리 5개(doppler-sim, constellation-sim, link-budget-calc,
kepler-orbit-sim, fading-channel-sim), 머신러닝과_인공지능 카테고리 3개(gradient-descent-lab,
cnn-conv-lab, bayes-inference-lab). 포트폴리오 다양성을 위해 **이번에는 위성통신과
머신러닝과_인공지능을 선택하지 않는다.** 아래 4개 후보 중에서 고른다.

## 후보 1: OS (원본: OS_소스.tex)
운영체제 역사·계보. 배치처리→멀티프로그래밍→유닉스 계보(BSD/GNU/Linux)→개인용 OS 전쟁(DOS/Mac/Windows)→
모바일(iOS/Android)→오늘날 시장 지형(리눅스 서버/클라우드 점유율, HarmonyOS 등). 수식보다 역사/계보 중심.

## 후보 2: 해석역학과_장이론 (원본: 해석역학_소스.tex)
달랑베르 원리→오일러-라그랑주 방정식→최소작용원리→해밀턴 정준형식→푸아송 괄호→슈뢰딩거 방정식→노터의 정리→
맥스웰 방정식→특수/일반상대성이론(측지선, 아인슈타인 방정식, 슈바르츠실트 해)→양자장론(파인만 다이어그램).

**참고**: 이 카테고리는 과거에 한 번 검토되어 `/home/hugok/MY-BLOG/spec-least-action-lab.md`에
"최소작용 원리 실험실"(1장 최소작용원리, 오일러-라그랑주 방정식) spec이 이미 작성된 적이 있다(사용자가
당시 머신러닝으로 방향을 바꿔 미구현 상태로 남음). 이번에 이 카테고리를 다시 선택해도 되고, 다른
후보를 선택해도 된다 — 자유롭게 판단하되, 만약 해석역학과_장이론을 선택한다면 그 spec을 그대로
재사용할지, 다른 장(예: 노터의 정리, 슈뢰딩거 방정식, 특수상대성이론 측지선 등)으로 새로 기획할지도
직접 원문을 재검토해 판단할 것(단순히 과거 spec을 복사하지 말고, 최선의 선택인지 다시 검증할 것).

## 후보 3: 물리전자와_반도체공학 (원본: 없음, 해당 폴더 CLAUDE.md 참고)

## 작업 절차

1. `/mnt/c/Users/hugok/Claude/Projects/doc/{후보폴더}/` 안의 최종 tex/pdf 소스를 실제로 읽어 확인한다
   (위 요약은 참고용 개요일 뿐, 세부 수식/개념은 원본에서 직접 확인할 것. 폴더 안에 자체 CLAUDE.md가 있으면 함께 읽는다).
2. 4개 후보 중 **가장 교육적이고 인터랙티브한 웹앱을 만들 수 있는** 하나를 선택한다
   (단순 텍스트 요약이 아니라, 사용자가 파라미터를 조작하며 직관을 얻을 수 있는 시각화/시뮬레이터 형태를 우선).
3. 선택한 폴더 안에서 구체적으로 어떤 개념을 웹앱으로 만들지 정한다
   (예: 유닉스 계보 트리, 라그랑주 역학 진자 시뮬레이터, PN접합 밴드다이어그램, 데데킨트 절단 시각화 등
   — 예시일 뿐 원본 확인 후 직접 판단).

## 출력 요구사항

`/home/hugok/MY-BLOG/spec-{앱이름-slug}.md` 파일을 다음 항목으로 작성한다 (앱이름은 하이픈-소문자 kebab-case로 정한다):

1. **선택한 카테고리와 웹앱** — 카테고리명(위 4개 중 하나, `data-category`에 그대로 쓸 것), 앱 이름(=폴더명, kebab-case), 한 줄 설명
2. **핵심 기능 목록** (5개 이내)
3. **파일 구조** — /apps/{앱이름}/ 하위 파일 목록
4. **주요 UI 구성** — 어떤 입력/출력/그래프가 있는지
5. **사용할 기술** — 순수 HTML/CSS/JS, 벤더링할 라이브러리(예: Chart.js — 기존 관례상 CDN보다 `js/vendor/`에
   로컬 복사 선호, 필요시 `apps/bayes-inference-lab/js/vendor/` 등 기존 사본 재사용 가능한지 확인)
6. **구현 범위 외** — 이번 버전에서 제외할 기능

spec 파일 작성 후 완료를 보고한다. **구현은 하지 않는다.** /home/hugok/MY-BLOG/index.html, css/, js/,
apps/ 하위 기존 폴더 등 다른 파일은 건드리지 않는다.
