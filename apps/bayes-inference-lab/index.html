베이즈 추론 실험실

암 검사의 기저율 오류, 오션뷰 호텔의 생성적 분류(MAP), 그리고 숫자게임으로 보는 MLE와 MAP의 갈림길까지 — 베이즈 정리가 실제 숫자에서 어떻게 작동하는지 슬라이더와 클릭으로 직접 확인합니다.

1. 기저율 오류 — 베이즈 정리 계산기

암 검사의 민감도 p(양성|암), 위양성률 p(양성|암 아님), 사전확률 p(암)을 조절하면 전체 확률의 법칙과 베이즈 정리로 p(암|양성)이 실시간으로 재계산됩니다. 민감도가 80%로 높아도 암의 사전확률(기저율)이 낮으면 양성 판정을 받은 사람이 실제 암일 확률은 여전히 낮을 수 있습니다 — 기본값(0.8, 0.096, 0.01)에서는 약 7.8%입니다.

민감도 p(양성|암) = 0.800
위양성률 p(양성|암 아님) = 0.096
사전확률 p(암) = 0.010
100명 아이콘 격자 — 100명 중 각 유형의 인원수
p(암|양성)

2. 조건부확률 방향성 — 성적표 예제

2×2 분할표(열심히 공부함/안 함 × A+ 받음/그 외)의 4칸을 직접 수정하면 p(A+|공부함)과 p(공부함|A+)을 나란히 계산합니다. p(A|B)와 p(B|A)는 서로 다른 질문이며, 표의 숫자에 따라 우연히 같을 수도 다를 수도 있습니다.

열심히 공부함
공부 안 함

p(공부함 | A+)

3. 생성적 분류기 — 오션뷰 호텔 MAP 분류

클래스(호텔)별 우도 p(오션뷰|y=c)와 사전확률 p(y=c)를 조절하면 베이즈 정리로 정규화된 사후확률이 즉시 갱신됩니다. 사후확률이 가장 큰 클래스가 MAP(최대 사후확률) 선택입니다 — 기본값에서는 호텔 A(사후확률 50%)가 선택됩니다.

4. 숫자게임(Number Game) 가설공간 탐색기

1~100 격자에서 숫자를 클릭해 데이터 D에 추가/제거하면, 고정된 가설공간 중 D와 모순되지 않는 가설들의 강한 표집 우도 p(D|h)=(1/|h|)^N을 계산해 비교합니다. D={16}일 때는 "2의 거듭제곱"이 근소 우세하지만, D={16,8,2,64}로 데이터가 늘면 우도비가 약 4800:1까지 벌어집니다.

1~100 숫자 격자 — 클릭으로 D에 추가/제거
가설별 p(D|h) — 로그 스케일, 모순 가설은 목록에서만 표시

5. MLE ↔ MAP 갈림길

"2의 거듭제곱"(h_two, |h|=6, 자연스러운 가설)과 "32를 제외한 2의 거듭제곱"(h', |h|=5, 조작적이고 부자연스러운 가설)을 비교합니다. 우도만 보면(MLE) h'이 더 간결한 가설공간 덕에 항상 유리하지만, h_two에 더 높은 사전확률(자연스러움 가중치)을 주면 사후확률(MAP)은 작은 N에서 h_two가 역전합니다. N을 늘리면 우도 항이 사전 항을 압도해 결국 MAP도 MLE와 같은 결론(h')으로 수렴합니다.

데이터 개수 N = 4
p(h_two) 사전확률 = 0.100
p(h') 사전확률 = 0.001
p(h_even) 사전확률 = 0.050

베이즈 추론 실험실 — 반복 최적화 없이 닫힌 형태 공식만으로 베이즈 정리, 생성적 분류, 숫자게임의 우도/사전확률 계산을 시연합니다.
