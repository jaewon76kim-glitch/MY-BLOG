### 후보 재검토 메모

- **SVM 커널트릭**: 여전히 쌍대문제 QP를 실시간으로 풀어야 하므로(SMO 단순화조차 수렴 안정성 이슈가 있음) 기각 사유가 유효하다고 판단, 이번에도 보류.
- **k-평균/GMM 애니메이션**: 웹에 유사 사례가 많다는 지적은 여전히 유효.
- **어텐션 가중치 시각화**: 재검토 결과, 사용자가 직접 토이 토큰 벡터와 $W_Q,W_K,W_V$를 입력해 실제 $QK^T/\sqrt{d_k}$ 계산을 보여주는 방식이라면 "학습된 임베딩 없이도" 메커니즘 자체는 정확히 시연 가능하다고 판단했다. 다만 이 경우 결과물이 "실제 문장에 대한 통찰"보다는 "소프트맥스 정규화와 스케일링의 수학적 시연"에 머물러, 교재가 강조하는 병목 해소·의미적 관련성이라는 핵심 서사를 전달하기 어렵다는 문제가 여전히 남는다. 최종적으로 보류.
- **퍼셉트론/역전파(7부) MLP+XOR 플레이그라운드**: 실제로 유력하게 검토했다. XOR 데이터에 대해 작은 MLP를 실시간 역전파로 학습시키고 결정경계가 바뀌는 과정을 애니메이션으로 보여주는 안이었다. 교육적 가치는 높으나, "시작점/학습률을 조절하며 최적화가 진행되는 것을 지켜본다"는 상호작용 패턴이 이미 구현된 `gradient-descent-lab`(경사하강 경로 애니메이션, η 슬라이더, 손실 감소 곡선)과 상당히 겹친다고 판단해 이번에는 제외하고, 8부(CNN)를 선택했다. CNN은 학습(training) 없이 순전파(forward pass)만으로도 핵심 개념(합성곱, 파라미터 효율성, 잔차연결)을 정확하게 시연할 수 있어 상호작용 방식이 근본적으로 다르다.

## 1. 앱 이름 및 한 줄 설명

**폴더명**: `apps/cnn-conv-lab/`

**한 줄 설명**: 완전연결층이 이미지에서 왜 비효율적인지부터, 합성곱·스트라이드·패딩·풀링이 어떻게 파라미터를 아끼는지, $1\times1$ 합성곱과 잔차연결이 각각 무엇을 해결하는지까지 — 실제 숫자를 직접 조작하며 눈으로 확인하는 CNN 메커니즘 실험실.

## 2. 왜 이 주제인가 (선정 이유)

교재 8부(`합성곱 신경망`)는 CNN을 "완전연결의 낭비 → 부분연결+가중치공유 → 합성곱 연산" 순서로 유도하고, 이어서 스트라이드/패딩/풀링의 하이퍼파라미터, LeNet-5의 파라미터 수 계산(156개 vs 480만개), GoogLeNet의 $1\times1$ 합성곱을 이용한 연산량 절감(726만회 vs 45만회), ResNet의 잔차연결이 기울기 소실을 막는 원리(곱셈 경로 + 덧셈 지름길)까지, **구체적인 숫자로 검증 가능한 예제**를 촘촘하게 제시한다. 이 예제들은 모두 학습(training) 없이도 순전파 계산만으로 정확하게 재현할 수 있다는 공통점이 있다.

1번째 앱(`gradient-descent-lab`)과 겹치지 않는 이유:
- **도메인이 다르다**: 회귀의 손실함수 등고선(가중치 공간)이 아니라, 이미지 공간에서의 커널 슬라이딩과 특징맵 생성을 다룬다.
- **상호작용 패턴이 다르다**: 1번째 앱은 "시작점을 찍고 최적화가 반복되며 수렴하는 과정"을 지켜보는 애니메이션이 핵심이었다면, 이번 앱은 "파라미터(커널 크기/스트라이드/패딩/채널 수)를 바꾸면 출력과 비용이 즉시 어떻게 달라지는가"를 탐색하는 계산기/시뮬레이터에 가깝다. 학습 반복(iteration) 개념 자체가 등장하지 않는다.
- **구현 리스크가 낮다**: 두 앱 모두 "닫힌 형태 수식만으로 정확히 계산 가능"하다는 원칙을 공유하지만, 이번 앱은 역전파나 최적화 수렴을 다루지 않고 순전파 연산(합성곱, 파라미터 개수 공식, 연산량 공식)만 다루므로 수치적으로 더 단순하고 안정적이다.
- 아직 다뤄지지 않은 신경망 계열(퍼셉트론/역전파, CNN, RNN/LSTM, 어텐션, LLM) 중에서 CNN을 선택한 이유는 위 "후보 재검토 메모" 참조.

## 3. 핵심 기능 목록 (5개 이내)

1. **합성곱 슬라이딩 데모**: 작은 그레이스케일 토이 이미지(예: $10\times10$ 정도, 픽셀값이 격자 위에 숫자로도 표시됨. 사용자가 직접 셀을 클릭해 밝기를 조절할 수 있음)와 $3\times3$ 커널(엣지 검출/블러/샤픈 프리셋 또는 사용자가 9칸 직접 입력하는 커스텀 커널)을 골라, 커널이 이미지 위를 슬라이딩하며 특징맵을 만드는 과정을 애니메이션으로 보여준다. 현재 커널 위치의 원소별 곱-합(dot product) 계산식을 숫자로 함께 표시한다(교재 8부 §1의 부분연결+가중치공유 설명을 그대로 시각화).
2. **스트라이드/패딩/풀링 파라미터 탐색기**: 커널 크기 $h$, 스트라이드 $s$, 패딩 $p$ 슬라이더를 조절하면 출력 특징맵 크기 $n' = \lfloor (n+2p-h)/s \rfloor + 1$ 공식이 실시간으로 재계산되고, 특징맵 캔버스 크기도 즉시 갱신된다(교재의 $n=11,h=3,s=1,p=0 \to n'=9$ 예시를 슬라이더 초기값으로 재현). 최대풀링/평균풀링 토글로 다운샘플링 결과도 비교한다.
3. **파라미터 수 비교 계산기(완전연결 vs 합성곱)**: 입력 크기, 은닉/출력 채널 수, 커널 크기를 입력하면 "완전연결로 연결했을 때"와 "합성곱(부분연결+가중치공유)으로 연결했을 때"의 파라미터 개수를 나란히 계산해 막대그래프로 비교한다. 교재의 LeNet-5 예시(입력 $32\times32$, $5\times5$ 커널 6개 → 156개 vs 완전연결 약 480만개)를 기본값으로 제공한다.
4. **$1\times1$ 합성곱 연산량 절감 계산기**: 입력 채널 수, 출력 채널 수, 커널 크기, 병목(bottleneck) 채널 수를 조절하면 "바로 큰 커널 적용" 대 "$1\times1$로 채널을 줄인 뒤 큰 커널 적용"의 곱셈 연산 횟수를 비교한다(교재의 $120\to20$채널, $5\times5$ 커널 예시: 726만 vs 45만을 기본값으로 재현). GoogLeNet 인셉션 모듈의 동기를 숫자로 체감시킨다.
5. **잔차연결(ResNet) 기울기 흐름 비교**: 층 수 $L$을 슬라이더로 조절하고, 각 층의 평균 기울기 인자(0~1 사이, 시그모이드 미분 최대값 등 프리셋 제공)를 설정하면, "일반 신경망"(순수 곱셈 경로, $\prod f'$)과 "잔차연결 포함"(곱셈 경로 + 덧셈 지름길 1)의 $\partial x_L/\partial x_1$ 값이 층수에 따라 어떻게 달라지는지 라인 차트로 비교한다(교재 8부 §4 수식 $\partial x_4/\partial x_1 = (\text{곱셈 경로}) + 1$을 일반화). 일반 신경망은 지수적으로 0에 수렴하지만 잔차연결은 항상 1 이상을 유지함을 시각적으로 확인시킨다.

## 5. 주요 UI 구성

- **공통 레이아웃**: 세로로 스크롤하는 5개 섹션(각각 위 핵심 기능 1~5에 대응), 상단에 섹션 이동용 앵커 네비게이션 바(모바일에서는 드롭다운 또는 가로 스크롤 탭으로 축소).
- **섹션 1 (합성곱 슬라이딩 데모)**:
  - 좌측: 토이 이미지 그리드(클릭으로 픽셀 밝기 조절, 0~255 값을 셀에 숫자로 표시)
  - 중앙: 커널 선택(프리셋 드롭다운: 엣지 검출/블러/샤픈/항등 + "커스텀" 선택 시 $3\times3$ 입력칸 노출)
  - 우측: 실시간 생성되는 특징맵 캔버스 + 현재 슬라이딩 위치의 dot product 계산식 텍스트
  - 하단: 재생/일시정지/스텝 버튼(슬라이딩 애니메이션 속도 조절)
- **섹션 2 (하이퍼파라미터 탐색기)**: 커널 크기 $h$, 스트라이드 $s$, 패딩 $p$ 슬라이더 + 입력 크기 $n$ 입력칸, 공식 $n'=\lfloor(n+2p-h)/s\rfloor+1$을 수식과 계산된 숫자로 함께 표시, 입력/출력 특징맵을 나란히 canvas로 시각화(패딩은 회색 테두리로 표시), 최대/평균풀링 토글.
- **섹션 3 (파라미터 수 비교)**: 입력 크기·채널 수·커널 크기·출력 채널 수 입력 폼, 결과를 Chart.js 막대그래프(완전연결 vs 합성곱, 로그 스케일 축 고려)로 비교, LeNet-5 예시를 불러오는 "예시 불러오기" 버튼.
- **섹션 4 ($1\times1$ 합성곱 계산기)**: 입력 채널/출력 채널/커널 크기/병목 채널 수 입력 폼, "직접 적용" vs "$1\times1$ 병목 후 적용" 두 경로의 연산 횟수를 나란히 숫자와 막대로 비교, GoogLeNet 예시 불러오기 버튼.
- **섹션 5 (잔차연결 기울기 흐름)**: 층 수 $L$ 슬라이더, 평균 기울기 인자 슬라이더(0.1~1.0), Chart.js 라인차트로 "잔차연결 없음"과 "잔차연결 있음" 두 곡선을 층수에 따라 비교(y축 로그 스케일 옵션).
- **모바일 대응**: 각 섹션의 좌/중/우 또는 좌/우 배치는 좁은 화면에서 세로로 스택.

## 6. 사용할 기술

- **기본**: 순수 HTML/CSS/JavaScript. 이미지 그리드, 커널 슬라이딩, 특징맵은 canvas 2D API로 직접 렌더링.
- **차트**: `apps/gradient-descent-lab/js/vendor/chart.umd.min.js`에 이미 로컬 벤더링된 Chart.js 사본을 그대로 복사해 재사용(섹션 3의 파라미터 수 막대그래프, 섹션 5의 기울기 흐름 라인차트). 신규 CDN 추가나 새 라이브러리 벤더링 불필요.
- **연산**: 모든 계산(합성곱 dot product, 출력 크기 공식, 파라미터 개수, 연산량, 기울기 곱)이 스칼라/소규모 행렬 연산으로 충분하므로 별도 선형대수 라이브러리 불필요.

## 7. 구현 범위 외

- 실제 학습(역전파)을 통한 커널 가중치 최적화 — 이 앱은 순전파(forward pass) 메커니즘 시연에 집중하고, 학습 과정은 다루지 않는다(학습 애니메이션은 이미 `gradient-descent-lab`에서 다룬 상호작용 패턴이므로 중복 회피).
- 실제 이미지 파일 업로드/드래그앤드롭 — 토이 이미지 그리드(픽셀 클릭 편집)로 한정하고, 사진 파일 처리(디코딩, 리사이즈 등)는 포함하지 않는다.
- LeNet-5/AlexNet/VGG/GoogLeNet/ResNet 전체 아키텍처의 층별 순전파 재현 — 각 구조에서 등장하는 핵심 개념(파라미터 효율성, $1\times1$ 합성곱, 잔차연결)만 발췌해 계산기로 보여주고, 전체 네트워크를 층층이 시뮬레이션하지는 않는다.
- 실제 합성곱 신경망을 브라우저에서 학습/추론하는 기능(TensorFlow.js 등 ML 프레임워크 도입) — 외부 ML 라이브러리를 쓰지 않고 직접 구현 가능한 범위(순수 수식 계산)로 한정한다.
- 배치정규화, 드롭아웃 등 정규화 기법의 별도 시각화 — 8부 본문에서 상세히 다루지 않으므로 이번 버전에서는 제외한다.
